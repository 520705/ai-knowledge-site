# 一、深度学习发展历史

## 1.1 起源与早期发展

深度学习的起源可以追溯到**1943年**，McCulloch和Pitts提出的MP神经元模型，首次将生物神经元数学化。1958年，Rosenblatt发明了**感知机（Perceptron）**，这是第一个具有学习能力的机器学习模型。

然而，1969年Minsky和Papert在《感知机》一书中证明了单层感知机的局限性——无法解决XOR问题，这导致了第一次**AI寒冬**。那时候的人们就像发现了汽车就认为马车要被淘汰一样兴奋，结果发现造出来的汽车还是个只能在平地上跑的怪物。

## 1.2 第二次浪潮与BP算法

1986年，Rumelhart、Hinton和Williams提出了**反向传播算法（Backpropagation）**，解决了多层神经网络的训练问题。这一时期，LeCun在手写数字识别方面取得了突破性进展，CNN雏形诞生。反向传播的出现就像给深度学习装上了发动机，虽然那时候的"深度"也就两三层的水平。

## 1.3 第三次浪潮与深度学习复兴

2006年，Hinton提出了**深度信念网络（DBN）**和"深度学习"概念，通过逐层预训练解决了深层网络训练困难的问题。2012年，AlexNet在ImageNet竞赛中以压倒性优势夺冠，标志着深度学习时代的正式到来。这一年堪称深度学习界的"iphone时刻"——从这一年起，整个AI领域开始翻天覆地。

2017年Transformer架构的提出更是将这场革命推向了新的高度。现在的GPT、Claude这些大模型，背后都是Transformer在撑腰。可以说，搞懂了Transformer，就搞懂了当代深度学习的半壁江山。

# 二、神经网络基础

## 2.1 神经元模型

神经元是神经网络的基本单元，它的灵感来自于人脑中的神经细胞。你可以把它想象成一个小工厂：接收原材料（输入），按照一定规则加工（加权求和），然后决定产品是否合格可以出货（激活函数）。

神经元的数学表达式为：

$$
y = f(\mathbf{w}^T \mathbf{x} + b) = f(\sum_{i=1}^{n} w_i x_i + b)
$$

其中：
- $\mathbf{x} = (x_1, x_2, ..., x_n)$ 是输入向量
- $\mathbf{w} = (w_1, w_2, ..., w_n)$ 是权重向量
- $b$ 是偏置项
- $f(\cdot)$ 是激活函数

打个比方的话，权重就像是不同原料的投料比例，偏置就像是基准产量要求，而激活函数则是决定产品是否达到出厂标准的质检环节。

```python
import numpy as np

class Neuron:
    def __init__(self, input_dim):
        # Xavier初始化权重 - 这是深度学习最常用的初始化方法之一
        # 初始化的学问很大，后面调参部分会详细讲
        self.w = np.random.randn(input_dim) * np.sqrt(2.0 / input_dim)
        self.b = 0
    
    def forward(self, x):
        z = np.dot(self.w, x) + self.b
        return self.activate(z)
    
    def activate(self, z):
        # ReLU激活函数 - 现在最流行的选择
        return np.maximum(0, z)
```

## 2.2 激活函数

激活函数为网络引入非线性，是深度学习拟合任意函数的关键。打个不太恰当但很形象的比喻：没有激活函数的神经网络，就像是一个只会做加法的计算器——能干的事情非常有限；加上激活函数之后，它就变成了一个能处理复杂逻辑的计算器。

### 2.2.1 Sigmoid函数

$$
\sigma(x) = \frac{1}{1 + e^{-x}}
$$

Sigmoid把任何数值都压缩到0和1之间，特别适合表示概率。这也是它名字的由来——像字母S一样有弯曲。它的优点很明显：输出在(0,1)之间，可以直接当成概率用，逻辑回归最爱用它。但它的缺点也很致命：两端的梯度几乎为0，这就是著名的**梯度消失问题**。想象一下，你在一个很大很大的碗里滚一颗小球，小球滚到碗边的时候几乎感受不到坡度了——这就是梯度消失的感觉。

### 2.2.2 Tanh函数

$$
\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}
$$

Tanh可以看成是Sigmoid的升级版，它把输出范围扩大到了(-1,1)。好处是数据变成了零中心的，坏处是两端同样会梯度消失。在LSTM的记忆门等地方还能看到它的身影，但在现代网络中已经不再是主力了。

### 2.2.3 ReLU函数

$$
\text{ReLU}(x) = \max(0, x)
$$

这可能是深度学习历史上最重要的"hack"之一。它简单到令人发指：大于0就输出原值，小于等于0就输出0。但就是这么简单，它解决了Sigmoid和Tanh的梯度消失问题，计算也飞快。你可以把ReLU理解为一个电灯开关：通电就亮，不通电就灭，没有中间地带。

```python
def leaky_relu(x, alpha=0.01):
    """
    Leaky ReLU - ReLU的改进版
    解决了"死神经元"问题：当输入小于0时，给一个很小的斜率让梯度能流过去
    alpha通常设成0.01，但也可以调参
    """
    return np.where(x > 0, x, alpha * x)

def gelu(x):
    """
    Gaussian Error Linear Unit - Transformer时代的新宠
    GELU ≈ 0.5 * x * (1 + tanh(√(2/π) * (x + 0.044715x³)))
    GPT、BERT这些大模型都用的它
    """
    return 0.5 * x * (1 + np.tanh(np.sqrt(2/np.pi) * (x + 0.044715 * x**3)))
```

## 2.3 损失函数

损失函数衡量模型预测值与真实值的差距，你可以把它理解为老师批改作业时打的分数。模型训练的目标就是让这个分数越来越高（或者说，分数代表的"损失"越来越低）。

### 2.3.1 均方误差损失（MSE）

$$
\mathcal{L}_{\text{MSE}} = \frac{1}{n} \sum_{i=1}^{n}(y_i - \hat{y}_i)^2
$$

MSE是最直观的损失函数，它计算的是预测值和真实值差的平方的平均值。大错 penalty 更大——预测错了100，离真实值差得远，比预测错了10要糟糕得多。适合用在房价预测、年龄预测这类回归任务上。

### 2.3.2 交叉熵损失（Cross-Entropy）

$$
\mathcal{L}_{\text{CE}} = -\sum_{i=1}^{n} y_i \log(\hat{y}_i)
$$

交叉熵是分类任务的标配损失函数。想象你在掷飞镖，交叉熵惩罚的是"你明明应该投中红心，却投到了蓝圈"这种错误。跟Sigmoid配合使用还能避免梯度消失——这是它的隐藏福利。

### 2.3.3 对比损失与三元组损失

这两种损失主要用于度量学习，比如人脸识别。核心思想是"让相似的东西离得更近，让不相似的东西离得更远"。

```python
def contrastive_loss(anchor, positive, negative, margin=1.0):
    """
    对比损失：anchor和positive应该近，anchor和negative应该远
    margin是一个间隔阈值，控制"多远算够远"
    """
    pos_dist = np.sum((anchor - positive) ** 2)  # 相似样本的距离
    neg_dist = np.sum((anchor - negative) ** 2)  # 不相似样本的距离
    loss = np.maximum(0, pos_dist - neg_dist + margin)
    return loss
```

## 2.4 反向传播算法

反向传播是深度学习的核心算法，也是让神经网络真正能"学习"的关键。它的本质是微积分里的链式法则，通过一步步回传来计算每个参数对最终损失的影响。

### 2.4.1 链式法则

对于复合函数 $f(g(x))$：
$$
\frac{df}{dx} = \frac{df}{dg} \cdot \frac{dg}{dx}
$$

这就像一条生产线上的质检：最终产品出了问题，要倒回去查每一道工序谁的责任更大。反向传播就是用这个逻辑，从输出层一路查回输入层，计算每个参数应该往哪个方向调整。

### 2.4.2 反向传播流程

1. **前向传播**：计算各层激活值和损失。把数据从输入层一路送到输出层，得到预测结果。
2. **反向传播**：从输出层向输入层计算梯度。看看预测结果和真实标签差多少，然后把误差一层层传回去。
3. **参数更新**：使用优化器更新权重和偏置。根据算出来的梯度，朝着让损失变小的方向走一步。

```python
class LinearLayer:
    def __init__(self, input_dim, output_dim):
        self.w = np.random.randn(input_dim, output_dim) * 0.01
        self.b = np.zeros(output_dim)
        self.grad_w = None
        self.grad_b = None
    
    def forward(self, x):
        self.x = x
        return np.dot(x, self.w) + self.b
    
    def backward(self, grad_output):
        # grad_output: 上游传来的梯度
        self.grad_w = np.dot(self.x.T, grad_output)
        self.grad_b = np.sum(grad_output, axis=0)
        grad_input = np.dot(grad_output, self.w.T)
        return grad_input
```

# 三、常见网络架构

## 3.1 卷积神经网络（CNN）

CNN通过**卷积层**提取局部特征，是计算机视觉的基础架构。你可以把它想象成一个拿着放大镜在图像上移动的侦探，每到一个位置就提取一下那里有什么特征。浅层可能提取边缘、纹理这些简单特征，深层则能组合出更复杂的模式。

### 3.1.1 卷积操作

$$
y[i,j] = \sum_{m}\sum_{n} x[i+m, j+n] \cdot k[m,n]
$$

卷积本质上就是一种特殊的"滑动窗口加权求和"操作。卷积核（kernel）就是那个滑动窗口，里面的数字就是权重。卷积核在图像上滑动，每到一个位置就和窗口内的像素做加权求和。

```python
import numpy as np

def conv2d(image, kernel, stride=1, padding=0):
    """
    手动实现二维卷积
    image: 输入图像
    kernel: 卷积核
    stride: 步长 - 卷积核每步移动多少像素
    padding: 填充 - 在图像边缘补多少圈0
    """
    if padding > 0:
        image = np.pad(image, ((padding, padding), (padding, padding)), mode='constant')
    
    h, w = image.shape
    k_h, k_w = kernel.shape
    out_h = (h - k_h) // stride + 1
    out_w = (w - k_w) // stride + 1
    
    output = np.zeros((out_h, out_w))
    for i in range(0, h - k_h + 1, stride):
        for j in range(0, w - k_w + 1, stride):
            output[i//stride, j//stride] = np.sum(image[i:i+k_h, j:j+k_w] * kernel)
    return output

# 一个简单的边缘检测卷积核
edge_kernel = np.array([
    [-1, -1, -1],
    [-1,  8, -1],
    [-1, -1, -1]
])
```

### 3.1.2 CNN核心组件

| 组件 | 作用 | 参数 |
|------|------|------|
| 卷积层 | 提取局部特征 | 卷积核大小、步长、填充 |
| 池化层 | 降维、保持特征 | Max Pooling / Avg Pooling |
| 全连接层 | 整合特征、输出 | 权重矩阵 |

池化层就像图片压缩，把相邻的像素合并成更少的像素。Max Pooling取最大值（最显著的特征），Avg Pooling取平均值（更平滑的特征）。

### 3.1.3 经典CNN架构

- **LeNet-5** (1998)：首个成功商用的CNN，LeCun用它来识别邮政编码。手写数字识别领域的老前辈。
- **AlexNet** (2012)：深度学习复兴的里程碑，2012年ImageNet竞赛冠军，top-5错误率只有15.3%，远超第二名的26.2%。
- **VGGNet** (2014)：结构简洁，特点是用3x3的小卷积核堆叠，很深的网络（16-19层）效果反而比直接用大卷积核好。
- **ResNet** (2015)：残差连接解决梯度消失问题，可以让网络堆到152层。ResNet的出现标志着"越深越好"时代的到来。
- **EfficientNet** (2019)：复合缩放策略，同时调整网络的深度、宽度和分辨率，在效率和精度之间找到了很好的平衡。

## 3.2 循环神经网络（RNN）

RNN通过**隐藏状态**处理序列数据，具有"记忆"能力。你可以把它想象成一个阅读理解高手，边读文章边记关键信息，读到后面还能联系前面的内容。

### 3.2.1 RNN基本结构

$$
h_t = \tanh(W_{xh}x_t + W_{hh}h_{t-1} + b_h)
$$
$$
y_t = W_{hy}h_t + b_y
$$

每个时间步，RNN都会接收当前输入和上一步的隐藏状态，输出新的隐藏状态。这个隐藏状态就像是一份笔记，记录了到目前为止看到的所有信息。

```python
class SimpleRNN:
    def __init__(self, input_dim, hidden_dim, output_dim):
        self.whh = np.random.randn(hidden_dim, hidden_dim) * 0.01
        self.wxh = np.random.randn(hidden_dim, input_dim) * 0.01
        self.why = np.random.randn(output_dim, hidden_dim) * 0.01
        self.bh = np.zeros(hidden_dim)
        self.by = np.zeros(output_dim)
    
    def forward(self, x_sequence):
        T = len(x_sequence)
        h = np.zeros((T + 1, self.whh.shape[0]))
        y = np.zeros((T, self.why.shape[0]))
        
        for t in range(T):
            h[t] = np.tanh(
                np.dot(self.wxh, x_sequence[t]) + 
                np.dot(self.whh, h[t-1]) + 
                self.bh
            )
            y[t] = np.dot(self.why, h[t]) + self.by
        
        return y, h
```

### 3.2.2 RNN的问题

RNN虽然理论上能处理任意长度的序列，但实践中有两个致命问题：

- **梯度消失/爆炸**：想象一下你把一封信传了很多个人，每传一次信息就会失真一点。RNN处理长序列时，早期的信息会被"遗忘"。
- **计算效率低**：序列必须顺序处理，无法并行。电影得一帧帧看，不能同时看第一帧和第十帧。

## 3.3 长短期记忆网络（LSTM）

LSTM通过**门控机制**解决长期依赖问题。LSTM就像给RNN装上了一个记事本，可以主动决定要记住什么、忘记什么。

### 3.3.1 LSTM核心公式

$$
\begin{aligned}
f_t &= \sigma(W_f \cdot [h_{t-1}, x_t] + b_f) \quad \text{（遗忘门）} \\
i_t &= \sigma(W_i \cdot [h_{t-1}, x_t] + b_i) \quad \text{（输入门）} \\
\tilde{C}_t &= \tanh(W_C \cdot [h_{t-1}, x_t] + b_C) \quad \text{（候选记忆）} \\
C_t &= f_t \cdot C_{t-1} + i_t \cdot \tilde{C}_t \quad \text{（记忆更新）} \\
o_t &= \sigma(W_o \cdot [h_{t-1}, x_t] + b_o) \quad \text{（输出门）} \\
h_t &= o_t \cdot \tanh(C_t) \quad \text{（隐藏状态）}
\end{aligned}
$$

### 3.3.2 门控机制解读

> [!info] LSTM门控机制
> - **遗忘门**：决定保留多少之前的记忆。比如读到一个新的主语，之前的旧主语可能就该"忘记"了
> - **输入门**：决定加入多少新信息。新看到的重要细节要"记住"
> - **输出门**：决定输出多少当前记忆。不是所有记忆都要输出，要筛选

## 3.4 Transformer架构

Transformer完全基于**注意力机制**，彻底改变了深度学习的格局。2017年的论文《Attention is All You Need》只有8页，但它的影响是革命性的——现在几乎所有的大模型都是基于Transformer。

### 3.4.1 核心组件

```python
class MultiHeadAttention:
    def __init__(self, d_model, num_heads):
        assert d_model % num_heads == 0
        self.d_k = d_model // num_heads
        self.num_heads = num_heads
        
        # Query, Key, Value投影矩阵
        self.W_q = np.random.randn(d_model, d_model) * 0.01
        self.W_k = np.random.randn(d_model, d_model) * 0.01
        self.W_v = np.random.randn(d_model, d_model) * 0.01
        self.W_o = np.random.randn(d_model, d_model) * 0.01
    
    def scaled_dot_product_attention(self, Q, K, V, mask=None):
        """
        缩放点积注意力
        核心思想：Query去和Key匹配，得到注意力权重，然后用权重对Value加权求和
        """
        scores = np.dot(Q, K.T) / np.sqrt(self.d_k)  # 缩放防止点积过大
        if mask is not None:
            scores = scores + mask  # mask用于遮盖不该看到的位置
        
        attn_weights = self.softmax(scores, axis=-1)
        return np.dot(attn_weights, V)
```

### 3.4.2 Transformer优势

- **并行计算**：摆脱RNN的顺序依赖，GPU可以同时计算所有位置的注意力
- **长距离依赖**：直接建模任意位置间的关系，不需要像RNN那样一步步传递
- **可扩展性**：便于堆叠深层网络，GPT-3就是堆了96层

## 3.5 生成对抗网络（GAN）

GAN通过**对抗训练**生成逼真数据，由Ian Goodfellow在2014年提出。它的核心思想是让两个网络互相博弈：生成器努力生成假数据骗过判别器，判别器努力识别假数据。两者斗智斗勇，最终生成器学会了生成以假乱真的数据。

### 3.5.1 GAN架构

$$
\min_G \max_D \mathbb{E}_{x \sim p_{data}}[\log D(x)] + \mathbb{E}_{z \sim p_z}[\log(1 - D(G(z)))]
$$

这个公式看起来复杂，其实就是一个二人零和博弈：判别器D要最大化目标，生成器G要最小化目标。两者轮流更新，谁也不让谁。

```python
class GAN:
    def __init__(self, latent_dim, data_dim):
        self.latent_dim = latent_dim
        self.generator = self.build_generator()
        self.discriminator = self.build_discriminator()
    
    def build_generator(self):
        """生成器：从随机噪声生成假数据"""
        model = Sequential([
            Dense(128, input_dim=self.latent_dim),
            LeakyReLU(0.2),  # 用LeakyReLU防止死神经元
            Dense(256),
            LeakyReLU(0.2),
            Dense(data_dim, activation='tanh')  # 输出归一化到[-1, 1]
        ])
        return model
    
    def build_discriminator(self):
        """判别器：判断数据是真是假"""
        model = Sequential([
            Dense(256, input_dim=data_dim),
            LeakyReLU(0.2),
            Dense(128),
            LeakyReLU(0.2),
            Dense(1, activation='sigmoid')  # 输出是真是假的概率
        ])
        return model
```

### 3.5.2 GAN训练常见问题

GAN训练出了名的"炼丹"：模式崩溃（生成样本多样性差）、训练不稳定、判别器太强把生成器压死……这些问题没有银弹，只能靠经验和一些trick。

| 变体 | 特点 | 应用场景 |
|------|------|----------|
| DCGAN | 深度卷积GAN | 图像生成 |
| WGAN | Wasserstein距离 | 稳定训练 |
| StyleGAN | 风格控制 | 人脸编辑 |
| Conditional GAN | 条件生成 | 类别控制 |

## 3.6 扩散模型（Diffusion Model）

扩散模型是近年来最火热的生成模型技术，Stable Diffusion、DALL-E 3、Sora都基于它。它的核心思想其实很符合物理：先把数据慢慢加噪变成纯噪声，再慢慢去噪恢复出数据。训练时模型学会"去噪"，推理时从纯噪声开始一步步去噪就能生成新数据。

### 3.6.1 扩散过程

**前向过程（加噪）**：
$$
q(x_t | x_{t-1}) = \mathcal{N}(x_t; \sqrt{1-\beta_t}x_{t-1}, \beta_t I)
$$

**反向过程（去噪）**：
$$
p_\theta(x_{t-1} | x_t) = \mathcal{N}(x_{t-1}; \mu_\theta(x_t, t), \Sigma_\theta(x_t, t))
$$

### 3.6.2 主流扩散模型

- **DDPM**：基础扩散模型，2020年提出，证明了扩散模型的可行性
- **Stable Diffusion**：latent diffusion，在压缩的 latent 空间生成，大幅降低计算成本
- **DALL-E 3**：结合CLIP的文本到图像生成，能更好地理解复杂文本
- **Sora**：OpenAI的视频生成扩散模型，能生成长达60秒的视频

# 四、训练技巧

## 4.1 Batch Normalization

BatchNorm通过**归一化层输入**来加速训练。每一层的输入数据分布可能差异很大，BatchNorm把它们都归一化到同一个分布，让后面的层能更稳定地学习。

$$
y = \frac{x - \mathbb{E}[x]}{\sqrt{\text{Var}[x] + \epsilon}} \cdot \gamma + \beta
$$

最后还有个$\gamma$和$\beta$参数负责scale和shift，让模型自己学习要不要保持归一化后的分布。这是很聪明的一个设计——如果归一化反而有害，模型可以学回来。

```python
class BatchNorm:
    def __init__(self, dim, eps=1e-5, momentum=0.1):
        self.gamma = np.ones(dim)  # 可学习的缩放参数
        self.beta = np.zeros(dim)  # 可学习的平移参数
        self.eps = eps
        self.momentum = momentum
        self.running_mean = np.zeros(dim)
        self.running_var = np.ones(dim)
    
    def forward(self, x, training=True):
        if training:
            mean = np.mean(x, axis=0)
            var = np.var(x, axis=0)
            self.x_norm = (x - mean) / np.sqrt(var + self.eps)
            # 更新running stats - 推理时用
            self.running_mean = self.momentum * self.running_mean + (1-self.momentum) * mean
            self.running_var = self.momentum * self.running_var + (1-self.momentum) * var
        else:
            # 推理时用running统计量
            self.x_norm = (x - self.running_mean) / np.sqrt(self.running_var + self.eps)
        
        return self.gamma * self.x_norm + self.beta
```

> [!tip] BatchNorm使用注意事项
> - batch size 不能太小，否则统计量不准，一般建议至少16
> - 训练和推理行为不同，切换时要调 mode
> - 对序列任务效果一般，不如 Layer Norm

## 4.2 Dropout

Dropout通过**随机丢弃神经元**防止过拟合。训练时随机"关掉"一些神经元，让剩下的神经元不能太依赖彼此，必须各自都学点东西。

```python
def dropout(x, p=0.5, training=True):
    """
    Dropout实现
    p: 丢弃概率
    training: 训练/推理标记
    """
    if training:
        # 生成掩码，1-p的概率为1，p的概率为0
        mask = np.random.binomial(1, 1-p, x.shape) / (1-p)
        return x * mask
    return x  # 推理时不做dropout
```

> [!tip] Dropout使用注意事项
> - p 一般设 0.2-0.5，太大了欠拟合，太小了没效果
> - 推理时不需要dropout，但要和训练保持一致的scale（PyTorch自动处理）
> - 近年来很多人发现不用Dropout效果也很好，改用其他正则化手段

## 4.3 学习率调度

学习率是深度学习最重要的超参数之一。太大容易震荡不收敛，太小收敛太慢。学习率调度就是在训练过程中动态调整学习率，让训练又快又稳。

### 4.3.1 Step Decay

每隔固定epoch把学习率降低一个比例。比如每10个epoch降低一半。

```python
def step_decay(epoch, initial_lr=0.1, drop_rate=0.5, epochs_drop=10):
    return initial_lr * (drop_rate ** (epoch // epochs_drop))
```

### 4.3.2 Cosine Annealing

学习率先增后降，用余弦函数控制，曲线平滑很好看：

$$
\eta_t = \eta_{\min} + \frac{1}{2}(\eta_{\max} - \eta_{\min})(1 + \cos(\frac{t\pi}{T}))
$$

### 4.3.3 Warmup + Decay

训练初期线性预热（比如前1000步学习率从0慢慢升到目标值），然后按余弦衰减。大模型训练标配，很多论文都验证过这个策略效果好。

# 五、优化器

## 5.1 SGD（随机梯度下降）

$$
\theta_{t+1} = \theta_t - \eta \nabla_\theta \mathcal{L}
$$

SGD是最基础的优化器，公式简单：梯度指向哪里就往哪里走。优点是简单可控，缺点是收敛慢，容易陷入局部最优或者在平坦区域磨蹭。

## 5.2 SGD with Momentum

$$
\begin{aligned}
v_t &= \beta v_{t-1} + (1-\beta) \nabla_\theta \mathcal{L} \\
\theta_{t+1} &= \theta_t - \eta v_t
\end{aligned}
$$

动量项累积历史梯度方向，就像一个滚下山坡的球，有惯性助力，能加速收敛，也能穿过一些小的平坦区域。$\beta$一般设0.9。

## 5.3 Adam（Adaptive Moment Estimation）

Adam是现在最流行的优化器，它把动量思想和自适应学习率结合起来。维护两个移动平均：梯度的一阶矩（类比动量）和梯度的二阶矩（自动调整每个参数的学习率）。

```python
class Adam:
    def __init__(self, lr=0.001, beta1=0.9, beta2=0.999, eps=1e-8):
        self.lr = lr
        self.beta1 = beta1
        self.beta2 = beta2
        self.eps = eps
        self.m = {}  # 一阶矩估计（动量）
        self.v = {}  # 二阶矩估计（自适应学习率）
        self.t = 0
    
    def update(self, params, grads):
        self.t += 1
        for key in params:
            # 梯度的一阶矩估计（均值）- 类比动量
            self.m[key] = self.beta1 * self.m.get(key, 0) + (1-self.beta1) * grads[key]
            # 梯度二阶矩估计（方差）- 自适应学习率
            self.v[key] = self.beta2 * self.v.get(key, 0) + (1-self.beta2) * grads[key]**2
            
            # 偏差校正 - 初期m和v都接近0，需要校正
            m_hat = self.m[key] / (1 - self.beta1**self.t)
            v_hat = self.v[key] / (1 - self.beta2**self.t)
            
            params[key] -= self.lr * m_hat / (np.sqrt(v_hat) + self.eps)
```

## 5.4 AdamW（Adam with Weight Decay）

AdamW将L2正则化从梯度分离出来，效果优于Adam+L2组合。这个改进很小，但效果很明显，现在大模型训练基本都是用AdamW。

```python
def adamw_update(param, grad, m, v, t, lr, weight_decay, beta1=0.9, beta2=0.999, eps=1e-8):
    m = beta1 * m + (1-beta1) * grad
    v = beta2 * v + (1-beta2) * grad**2
    m_hat = m / (1 - beta1**t)
    v_hat = v / (1 - beta2**t)
    
    # 权重衰减独立于动量估计 - 这是AdamW和Adam+L2的区别
    param = param - lr * (m_hat / (np.sqrt(v_hat) + eps) + weight_decay * param)
    return param, m, v
```

## 6.1 迁移学习策略

迁移学习是深度学习最重要的实战技巧之一。预训练模型已经在大规模数据上学到了很多通用特征，我们要做的就是站在巨人的肩膀上，根据自己的任务做微调。

## 7.3 模型定义

这里我们用一个简洁但有效的CNN架构：

```python
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super(SimpleCNN, self).__init__()
        
        # 卷积层特征提取器
        self.conv1 = nn.Conv2d(3, 32, kernel_size=3, padding=1)
        self.bn1 = nn.BatchNorm2d(32)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.bn2 = nn.BatchNorm2d(64)
        self.conv3 = nn.Conv2d(64, 128, kernel_size=3, padding=1)
        self.bn3 = nn.BatchNorm2d(128)
        
        self.pool = nn.MaxPool2d(2, 2)
        self.dropout = nn.Dropout(0.5)
        
        # 全连接分类器
        self.fc1 = nn.Linear(128 * 28 * 28, 256)  # CIFAR-10图片是32x32，经过3次池化变成28x28
        self.fc2 = nn.Linear(256, num_classes)
    
    def forward(self, x):
        # Conv block 1: 32x32 -> 16x16
        x = self.pool(F.relu(self.bn1(self.conv1(x))))
        # Conv block 2: 16x16 -> 8x8
        x = self.pool(F.relu(self.bn2(self.conv2(x))))
        # Conv block 3: 8x8 -> 4x4
        x = self.pool(F.relu(self.bn3(self.conv3(x))))
        
        x = x.view(-1, 128 * 28 * 28)  # 展平
        x = self.dropout(x)
        x = F.relu(self.fc1(x))
        x = self.dropout(x)
        x = self.fc2(x)
        return x

model = SimpleCNN(num_classes=10).to(device)
print(model)
```

## 7.4 训练循环

训练一个深度学习模型，本质上就是一个不断迭代优化的过程：前向传播计算预测→计算损失→反向传播计算梯度→更新参数。

```python
import torch.optim as optim

# 损失函数和优化器
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=0.001, weight_decay=1e-4)
scheduler = optim.lr_scheduler.ReduceLROnPlateau(optimizer, mode='min', factor=0.5, patience=3)

def train_epoch(model, train_loader, criterion, optimizer, device):
    model.train()
    running_loss = 0.0
    correct = 0
    total = 0
    
    for batch_idx, (inputs, targets) in enumerate(train_loader):
        inputs, targets = inputs.to(device), targets.to(device)
        
        optimizer.zero_grad()  # 梯度清零，否则会累积
        outputs = model(inputs)
        loss = criterion(outputs, targets)
        loss.backward()  # 反向传播
        optimizer.step()  # 更新参数
        
        running_loss += loss.item()
        _, predicted = outputs.max(1)
        total += targets.size(0)
        correct += predicted.eq(targets).sum().item()
        
        # 每100个batch打印一次
        if (batch_idx + 1) % 100 == 0:
            print(f'  Batch {batch_idx+1}/{len(train_loader)}, '
                  f'Loss: {loss.item():.4f}, '
                  f'Acc: {100.*correct/total:.2f}%')
    
    return running_loss / len(train_loader), 100. * correct / total

def evaluate(model, test_loader, criterion, device):
    model.eval()
    test_loss = 0.0
    correct = 0
    total = 0
    
    with torch.no_grad():  # 推理时不计算梯度，速度快很多
        for inputs, targets in test_loader:
            inputs, targets = inputs.to(device), targets.to(device)
            outputs = model(inputs)
            loss = criterion(outputs, targets)
            
            test_loss += loss.item()
            _, predicted = outputs.max(1)
            total += targets.size(0)
            correct += predicted.eq(targets).sum().item()
    
    return test_loss / len(test_loader), 100. * correct / total
```

## 7.5 完整训练流程


# 八、调参炼丹手册

调参（也叫"炼丹"）是深度学习最有意思也最让人头疼的部分。这一章分享一些实战的调参经验和技巧。

## 8.1 学习率（Learning Rate）

学习率可能是最重要的超参数。我见过太多人模型不收敛，一查发现学习率设错了。

**经验法则**：

- Adam优化器：学习率设 1e-3 到 1e-4 比较保险
- SGD + Momentum：学习率设 0.01 到 0.1，用 momentum=0.9
- 学习率太高：loss爆炸变成NaN，或者loss振荡不收敛
- 学习率太低：训练太慢，可能困在局部最优

**实战技巧**：

1. **用学习率查找器（Learning Rate Finder）**：从很小的学习率开始，逐渐增大，记录loss变化，找到loss下降最快的区间
2. **用学习率调度**：训练初期用warmup稳定训练，后期用余弦衰减或step decay找到更好的局部最优
3. **差分学习率**：对预训练层用小学习率，对新层用大学习率

```python
# PyTorch实现学习率查找器
def find_learning_rate(model, train_loader, optimizer, criterion, device, init_lr=1e-7, max_lr=10, beta=0.98):
    n = 0
    avg_loss = 0
    best_loss = 0
    losses = []
    lrs = []
    
    optimizer.param_groups[0]['lr'] = init_lr
    
    for images, labels in train_loader:
        images, labels = images.to(device), labels.to(device)
        optimizer.zero_grad()
        outputs = model(images)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()
        
        n += 1
        avg_loss = beta * avg_loss + (1 - beta) * loss.item()
        smoothed_loss = avg_loss / (1 - beta ** n)
        
        if n > 1 and smoothed_loss > 10 * best_loss:
            break
        
        if smoothed_loss < best_loss or n == 1:
            best_loss = smoothed_loss
        
        losses.append(smoothed_loss)
        lrs.append(optimizer.param_groups[0]['lr'])
        
        optimizer.param_groups[0]['lr'] *= 1.1
    
    return lrs, losses
```

## 8.2 Batch Size

batch size影响的不只是显存占用，还会影响模型收敛速度和最终性能。

**经验法则**：

- 大 batch size（512-4096）：收敛更稳定，泛化能力可能稍差，适合大模型
- 小 batch size（16-128）：泛化能力可能更好，但收敛更抖动，需要更多epoch
- 受限于显存时，尽量选2的幂次：32, 64, 128...

**实战技巧**：

1. GPU显存不够？试试梯度累积——把多个小batch的梯度累在一起，等效大batch
2. BatchNorm对batch size敏感，太小可能导致统计量不准
3. 小batch训更久可能达到和大batch相同的性能，只是需要更多迭代

```python
# 梯度累积实现
accumulation_steps = 4  # 累积4个小batch
effective_batch_size = batch_size * accumulation_steps

optimizer.zero_grad()
for i, (images, labels) in enumerate(train_loader):
    images, labels = images.to(device), labels.to(device)
    outputs = model(images)
    loss = criterion(outputs, labels) / accumulation_steps  # 损失也除以累积步数
    loss.backward()
    
    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

## 8.3 正则化

正则化是防止过拟合的关键手段。数据不够、模型太大、训练太久都可能导致过拟合。

### 8.3.1 L2正则化（权重衰减）

L2正则化让权重倾向于取小值，降低模型复杂度。在AdamW中，权重衰减是独立于动量估计的，效果更好。

### 8.3.2 Dropout

如前所述，随机丢弃神经元防止共同适应。

### 8.3.3 Early Stopping

监控验证集loss，连续多个epoch不下降就停止训练，防止过度拟合训练数据。

```python
# Early Stopping实现
class EarlyStopping:
    def __init__(self, patience=10, min_delta=0.001):
        self.patience = patience
        self.min_delta = min_delta
        self.counter = 0
        self.best_loss = None
        self.should_stop = False
    
    def __call__(self, val_loss):
        if self.best_loss is None:
            self.best_loss = val_loss
        elif val_loss > self.best_loss - self.min_delta:
            self.counter += 1
            if self.counter >= self.patience:
                self.should_stop = True
        else:
            self.best_loss = val_loss
            self.counter = 0
        return self.should_stop
```

## 8.4 初始化

权重初始化对深层网络的训练至关重要。初始化太大会梯度爆炸，太小会梯度消失。

**常用初始化方法**：

- **Xavier初始化**：适合Sigmoid和Tanh
- **He初始化**：适合ReLU及其变体
- **预训练初始化**：用在大模型预训练权重上微调

```python
# PyTorch中各层默认使用合适的初始化
# 如果需要手动初始化
for m in model.modules():
    if isinstance(m, nn.Conv2d):
        nn.init.kaiming_normal_(m.weight, mode='fan_out', nonlinearity='relu')
        if m.bias is not None:
            nn.init.constant_(m.bias, 0)
    elif isinstance(m, nn.BatchNorm2d):
        nn.init.constant_(m.weight, 1)
        nn.init.constant_(m.bias, 0)
    elif isinstance(m, nn.Linear):
        nn.init.normal_(m.weight, 0, 0.01)
        nn.init.constant_(m.bias, 0)
```

# 九、常见错误与排查

这一章汇总了深度学习实战中最常见的问题和解决方案。

## 9.1 梯度消失与梯度爆炸

这是两个极端——梯度太小让网络学不动，梯度太大让训练发散。

**症状识别**：

- 梯度消失：训练loss几乎不变，或者变得非常慢
- 梯度爆炸：loss变成NaN或Inf，或者loss剧烈振荡

**解决方案**：

1. **梯度裁剪**：限制梯度的最大范数，防止爆炸

```python
# 梯度裁剪
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
optimizer.step()
```

2. **选择合适的激活函数**：ReLU比Sigmoid/Tanh更不容易梯度消失
3. **残差连接**：ResNet的skip connection让梯度能直接回传
4. **合适的初始化**：He初始化对ReLU友好
5. **BatchNorm**：归一化让数据分布更稳定

## 9.2 过拟合

模型在训练集上表现很好，但在测试集上差很多。

**症状识别**：

- 训练loss持续下降，验证loss不再下降甚至上升
- 训练准确率接近100%，验证准确率差很远

**解决方案**：

1. **数据增强**：扩充训练数据的多样性
2. **正则化**：L2、Dropout、Early Stopping
3. **降低模型复杂度**：减少层数或参数量
4. **使用验证集调参**：不要用测试集调参，那是用来最终评估的
5. **Cross Validation**：数据少时用k折交叉验证

```python
# 检查过拟合程度
train_loss, train_acc = evaluate(model, train_loader, criterion, device)
test_loss, test_acc = evaluate(model, test_loader, criterion, device)
print(f'训练集准确率: {train_acc:.2f}%')
print(f'测试集准确率: {test_acc:.2f}%')
print(f'差距: {train_acc - test_acc:.2f}%')  # 差距太大说明过拟合严重
```

## 9.3 NaN和Inf问题

Loss变成NaN是最让人崩溃的错误之一。

**常见原因**：

1. **学习率太高**：梯度更新过大，导致数值溢出
2. **除零**：损失计算中除以了0
3. **梯度爆炸**：同上
4. **数据问题**：输入数据有异常值或NaN

**排查步骤**：

```python
# 在训练循环中加入调试代码
if torch.isnan(loss):
    print("Loss is NaN! Debugging...")
    print(f"Inputs min: {inputs.min()}, max: {inputs.max()}")
    print(f"Outputs min: {outputs.min()}, max: {outputs.max()}")
    print(f"Gradients before step:")
    for name, param in model.named_parameters():
        if param.grad is not None:
            print(f"  {name}: grad min={param.grad.min():.4f}, max={param.grad.max():.4f}")
    break
```

**解决方案**：

1. 先把学习率降低10倍试试
2. 检查数据归一化是否正确
3. 梯度裁剪
4. 给损失加一个小的epsilon防止除零
5. 检查数据管道有没有bug

## 9.4 显存不足（OOM）

这是深度学习最常见的硬件限制问题。

**解决方案**：

1. **减小batch size**：最直接的办法
2. **梯度累积**：如前所述
3. **混合精度训练**：用float16减少显存占用

```python
# PyTorch混合精度训练
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for images, labels in train_loader:
    images, labels = images.to(device), labels.to(device)
    
    optimizer.zero_grad()
    
    with autocast():
        outputs = model(images)
        loss = criterion(outputs, labels)
    
    scaler.scale(loss).backward()  # 缩放损失防止梯度下溢
    scaler.step(optimizer)
    scaler.update()
```

4. **checkpointing**：不保存中间激活值，recompute换取显存
5. **清理不需要的变量**：`del`, `torch.cuda.empty_cache()`
6. **用更小的模型**：换参数量更小的架构

## 9.5 模型不收敛

Loss不下降是最常见的问题之一。

**排查清单**：

```python
# 完整排查代码
def debug_training(model, train_loader, criterion, device):
    model.train()
    first_batch = True
    
    for images, labels in train_loader:
        images, labels = images.to(device), labels.to(device)
        
        # 检查数据
        print(f"Images - min: {images.min():.4f}, max: {images.max():.4f}, "
              f"mean: {images.mean():.4f}, std: {images.std():.4f}")
        print(f"Labels - unique: {torch.unique(labels)}")
        
        # 检查模型输出
        outputs = model(images)
        print(f"Outputs - min: {outputs.min():.4f}, max: {outputs.max():.4f}")
        
        # 检查损失
        loss = criterion(outputs, labels)
        print(f"Loss: {loss.item():.4f}")
        
        # 检查梯度
        loss.backward()
        total_grad_norm = 0
        for name, param in model.named_parameters():
            if param.grad is not None:
                grad_norm = param.grad.norm().item()
                total_grad_norm += grad_norm ** 2
                if first_batch:
                    print(f"  {name}: grad_norm={grad_norm:.6f}")
        total_grad_norm = total_grad_norm ** 0.5
        print(f"Total gradient norm: {total_grad_norm:.6f}")
        
        first_batch = False
        break  # 只看第一个batch
```

**常见原因和解决**：

| 问题 | 原因 | 解决 |
|------|------|------|
| Loss = NaN | 学习率太高、数据有NaN | 降低学习率、检查数据 |
| Loss不动 | 学习率太低、梯度传不回去 | 提高学习率、检查梯度 |
| Loss振荡 | 学习率太高、batch size太小 | 降低学习率、增大batch |
| Loss上升 | 学习率太高、梯度爆炸 | 降低学习率、梯度裁剪 |

## 10.3 推荐系统

深度学习在推荐系统里也有广泛应用，比如用神经网络学习用户和物品的embedding，然后做召回和排序。

**实战案例：电影推荐embedding**

```python
import torch
import torch.nn as nn

class RecommenderNet(nn.Module):
    def __init__(self, num_users, num_movies, embedding_size=50):
        super().__init__()
        self.user_embedding = nn.Embedding(num_users, embedding_size)
        self.movie_embedding = nn.Embedding(num_movies, embedding_size)
        self.fc = nn.Sequential(
            nn.Linear(embedding_size * 2, 64),
            nn.ReLU(),
            nn.Linear(64, 32),
            nn.ReLU(),
            nn.Linear(32, 1)
        )
    
    def forward(self, user_id, movie_id):
        user_vec = self.user_embedding(user_id)
        movie_vec = self.movie_embedding(movie_id)
        # 拼接用户和电影embedding，预测评分
        combined = torch.cat([user_vec, movie_vec], dim=-1)
        rating = self.fc(combined)
        return rating.squeeze()

# 用户想看什么电影？找相似embedding的电影
def recommend_movies(user_id, movie_ids, model, top_k=5):
    model.eval()
    with torch.no_grad():
        user_emb = model.user_embedding(user_id)
        movie_embs = model.movie_embedding(movie_ids)
        # 计算余弦相似度
        similarity = torch.cosine_similarity(user_emb, movie_embs)
        top_indices = similarity.topk(top_k).indices
    return [movie_ids[i].item() for i in top_indices]
```

# 十一、新手常见问题FAQ

## Q1: 为什么我的模型训练 loss 在下降但测试准确率很低？

这很可能就是过拟合了——模型把训练数据背下来了，但没学到真正的规律。检查一下：数据增强够不够？正则化强度够不够？模型是不是太大了？也可以用上面"检查过拟合程度"的代码看看训练集和测试集的差距。

## Q2: 显存不足怎么办？

首先试试减小batch size，如果还是不够就用梯度累积。然后考虑混合精度训练（float16），这基本能省一半显存。如果显存真的很紧张，可以试试更小的模型版本，或者用模型并行把模型分到多张卡上。

## Q3: 为什么用 ReLU 但还是梯度消失？

ReLU 只解决了正区的梯度消失问题，负区梯度为0会导致"死神经元"。试试 Leaky ReLU 或者 PReLU，或者检查一下网络结构是不是太深了，考虑加残差连接。

## Q4: 学习率设多少合适？

没有万能答案，但有一些经验：从 1e-3 开始试试（Adam）。如果 loss 不降或变NaN，降低10倍；如果下降太慢，增大10倍。也可以用学习率查找器找到最优点。

## Q5: 数据不够怎么办？

几个思路：一是数据增强，人为增加数据多样性；二是迁移学习，用预训练模型改的；三是合成数据，用GAN或扩散模型生成一些；四是自监督学习，在无标签数据上学表示。

## Q6: 训练到一半 loss 突然变大是怎么回事？

可能是学习率太高，或者遇到了梯度爆炸。试试降低学习率，或者加梯度裁剪。也可能是数据里出现了异常值，检查一下数据管道。

## Q7: 模型收敛了但准确率很低怎么办？

准确率低可能是模型太小、数据太难、或者特征不够信息量。先检查数据标签对不对，特征归一化对不对。然后试试更深更宽的模型，或者换更好的预训练模型。

## Q8: 多 GPU 训练怎么做？

PyTorch 的 DistributedDataParallel 是最常用的方案：

```python
import torch.nn.parallel
model = torch.nn.parallel.DistributedDataParallel(model, device_ids=[0,1,2,3])
```

也可以用更简单的 DataParallel，但效率稍低。Hugging Face Accelerate 库也提供了很好的封装。

# 十三、迁移学习

## 13.1 迁移学习策略

迁移学习是深度学习最重要的实战技巧之一。预训练模型已经在大规模数据上学到了很多通用特征，我们要做的就是站在巨人的肩膀上，根据自己的任务做微调。

| 策略 | 适用场景 | 微调层 |
|------|----------|--------|
| 特征提取 | 数据量小（几百张），相似任务 | 仅最后一层 |
| 浅层微调 | 数据量中等（几千张） | 底层+新层 |
| 全量微调 | 数据量充足（几万张以上） | 全部层 |

## 13.2 预训练模型来源

- **ImageNet预训练**：ResNet、VGG、EfficientNet等，在120万张图片上训练过
- **大规模预训练**：BERT、GPT、LLaMA等语言模型，在互联网文本上预训练
- **多模态预训练**：CLIP、BLIP、SAM等，跨越文本和图像

```python
import torchvision.models as models
import torch.nn as nn

# 特征提取模式 - 只训练新的分类头
model = models.resnet50(pretrained=True)
for param in model.parameters():
    param.requires_grad = False  # 冻结所有预训练层

# 只训练新的分类头
model.fc = nn.Linear(model.fc.in_features, num_classes)

# 微调模式 - 解冻部分层
for param in model.layer4.parameters():
    param.requires_grad = True  # 解冻最后一层block

# 优化器只优化requires_grad=True的参数
optimizer = torch.optim.Adam(filter(lambda p: p.requires_grad, model.parameters()), lr=0.001)
```
