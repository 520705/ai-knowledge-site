# 提示词工程从入门到精通

> **写给谁看**：完全不懂AI的普通人，想用AI帮忙干活、写文章、做分析
> 
> **看完能得到什么**：学会跟AI好好说话，让AI真正帮你解决问题
> 
> **更新日期**：2026年4月

---

## 一、先聊聊：什么是"提示词"，跟你有什么关系？

### 1.1 开门见山，提示词到底是啥？

你有没有遇到过这种情况：兴冲冲地打开ChatGPT或者其他AI工具，问了个问题，结果AI的回答牛头不对马嘴，气得你想把电脑砸了？

别急，这不是AI的问题，是你的问题——准确说，是你"提问方式"的问题。

**提示词（Prompt）**，简单来说就是你跟AI对话时说的话。你说的清楚，AI就回答得明白；你说的含糊，AI就给你整一堆废话。

举个生活中的例子你就懂了：

你去早餐店点餐：

- ❌ 你说："来个那个" → 老板一脸懵逼
- ✅ 你说："老板，来两个煎饼果子，一个加鸡蛋火腿，一个加生菜不要酱" → 老板秒懂，三分钟搞定

跟AI对话也是一样的道理。你说"帮我写篇文章"，AI不知道写什么主题、给谁看、多长、什么风格，当然只能给你一篇万能废话。你说"帮我写一篇2000字的职场成长类公众号文章，目标读者是25-30岁的职场新人，要有3个真实故事案例，文风轻松有温度，读完让人感觉被鼓励到了"，AI就能给你一份像样的初稿。

**所以，提示词工程（Prompt Engineering）说白了就是：研究怎么跟AI说话，让它听懂你的意思，干出你想要的活。**

### 1.2 为什么你用AI总觉得不好用？

我见过太多人用AI的状态：满怀期待地问了一个问题，看到回答后翻白眼，然后放弃。

问题出在哪里？我总结了三个最常见的原因：

**第一个原因：你把AI当人，但它不是人**

AI没有读心术。你脑子里想的那个"完美的答案"，AI是看不到的。你必须把你的需求完整、准确地说出来。越是具体的要求，AI完成得越好。

**第二个原因：你问的问题太开放了**

"你怎么看AI？" "帮我分析下市场？" 这种问题太抽象了，AI只能给你一个同样抽象的废话回答。你得把你的问题掰碎了、揉小了问。

**第三个原因：你不知道AI能干啥**

很多人用AI就是图个新鲜，问两个问题就跑了，根本没摸到AI真正强大的地方。AI真正厉害的是：帮你写各种材料、分析数据、做计划、给你当顾问、当你的学习助手……关键是你得知道怎么用它。

### 1.3 学会提示词工程有什么用？

说几个真实的场景：

**场景一：写工作总结**

以前：憋了半天写不出来，拖到deadline随便糊弄一篇。
现在：给AI一个框架，5分钟生成初稿，你改改就能交。

**场景二：学新东西**

以前：网上找教程，看到一半看不下去。
现在：让AI用你能听懂的方式解释，然后让它出题考你，不懂的地方追问。

**场景三：做决策**

以前：纠结来纠结去，想不清楚。
现在：把情况跟AI说清楚，让它帮你分析利弊，给出建议。

**场景四：写代码**

以前：遇到bug卡半天，百度搜半天看不懂。
现在：直接把错误信息扔给AI，它帮你分析原因，给出解决方案。

这些场景不是我瞎编的，是我每天都在用的真实体验。学会提示词工程，真的能帮你省大量时间，做更多事情。

## 二、原理篇：AI是怎么理解你说的话的？

### 2.1 先搞清楚AI的"脑回路"

这一部分可能会有点抽象，但你得了解一点基础原理，才能更好地驾驭AI。

现在的AI（主要是大语言模型，比如GPT、Claude、DeepSeek这些），它们的本质是一个超级大的"文字接龙"机器。

什么意思呢？就是你给AI一段文字，它会根据这段文字，预测下一个字最可能是什么，然后再预测下下一个字……一直预测下去，直到生成完整的回答。

这个预测不是胡猜的。AI在训练的时候，学习了海量的文本数据——书籍、网页、文章、代码……几十亿甚至上百亿的文字。它从这些数据里学到了：什么样的上下文，通常会接什么样的文字。

所以当你输入"今天天气真"的时候，AI预测下一个字大概率是"好"。当你输入"Python是一种"的时候，AI预测下一个字可能是"编程语言"。

**这个原理告诉我们一个关键道理：AI是根据你的输入来生成输出的。你的输入越清晰、越具体，AI的输出就越可能符合你的预期。**

### 2.2 AI的"优点"和"缺点"

了解AI的特点，你才能扬长避短：

**AI的优点：**

- 知识面超级广：读过的资料比你一辈子能读的还多
- 不知疲倦：007工作制也不会抱怨
- 速度快：几秒钟生成一篇文章，一分钟分析一份报告
- 可以扮演不同角色：可以假装是律师、医生、老师、营销专家……
- 会推理：能做数学题、分析逻辑、写代码debug

**AI的缺点：**

- 会"做梦"（也叫幻觉）：有时候会一本正经地胡说八道
- 不知道最新信息：训练数据有截止日期
- 没有真正的理解：它只是"像"在思考，其实是在玩文字游戏
- 容易被你带偏：如果你给了错误信息，它会顺着你的错误继续错

**所以，记住一个黄金法则：AI是你的工具，不是你的大脑。它的输出你要审核，不能照搬。涉及重要事实的内容，要核实。**

### 2.3 上下文越多，结果越准

这是提示词工程最核心的原理，没有之一。

AI的每一次回答，都是基于你给的"上下文"生成的。这个上下文包括：

- 你在提示词里告诉它的所有信息
- 你之前跟它对话的内容（在同一个对话窗口里）
- 你上传的文件或图片
- 你选择的AI模式或功能

**你给的上下文越多、越清晰，AI就越能准确理解你想要什么。**

这个道理就像你跟一个聪明但不了解你情况的助手合作：

- 你说："帮我分析下我们公司的问题" → 助手啥都不知道，只能泛泛而谈
- 你说："我们是一家做B2B软件的创业公司，目前50人，融资到A轮，月流水50万，主要问题是客户流失率高，上个月流失了20%，我已经做了用户访谈，发现主要原因是产品体验差、客服响应慢。请帮我分析下。" → 助手能给出真正有价值的建议

## 三、基础篇：写出好提示词的七种武器

### 3.1 武器一：说清楚你要什么

这是最重要的一条。

❌ 模糊说法：
- "帮我写篇文章"
- "分析下这个数据"
- "给我一些建议"

✅ 清晰说法：
- "帮我写一篇1500字的职场成长类公众号文章，目标读者是工作1-3年的年轻人，要有代入感，结尾要有行动召唤"
- "分析这份销售数据，找出增长最快的三个产品类别，计算每个类别的环比增长率，并用表格呈现"
- "我最近想转行，从传统行业转到互联网产品经理，目前有3年工作经验，请给我3个具体的转型建议，每个建议要包含具体的学习资源和时间规划"

### 3.2 武器二：说清楚你是什么人、给谁看

AI需要知道它的"角色"和"观众"，才能调整说话的方式。

❌ 没有角色：
- "解释下什么是区块链"
- （AI给出一堆技术术语，你看不懂）

✅ 有角色和受众：
- "我是一个完全不懂技术的小白，请用我能听懂的比喻，解释什么是区块链"
- "我是区块链开发工程师，需要向不懂技术的领导汇报区块链的应用前景，请写一份通俗易懂的汇报材料"
- "我是一个高三学生，正在准备自主招生面试，请用面试官喜欢听的方式，解释什么是区块链"

同一个问题，不同身份的人需要不同的回答。

### 3.3 武器三：给AI一个"身份"

让AI扮演一个角色，往往能大幅提升输出质量。

你可以这样开头：
- "你是一位有10年经验的产品经理"
- "你是一位资深的新媒体运营专家"
- "你是一位心理咨询师"
- "你是一位考研数学140分的学霸"

然后再加上你的具体问题。

**为什么给身份有效？** 因为AI在训练时学了很多不同身份的人说的话。告诉它"你是产品经理"，它就会调用"产品经理"这个角色的说话方式、专业知识和思考角度。

### 3.4 武器四：给出例子（Few-shot）

如果你能给出1-3个例子，AI往往能更准确地理解你的需求。

❌ 只给指令：
```
帮我写一个产品介绍文案
```

✅ 给例子：
```
帮我写一个产品介绍文案，参考这个风格：

【例子】
产品：某品牌保温杯
文案：冬日里的一杯热茶，温暖你的每一个早晨。某品牌保温杯，12小时保温保冷，医用级不锈钢内胆，轻巧便携。给自己，或送给你在乎的人。

---

现在帮我写一个产品介绍文案：
产品：某品牌无线蓝牙耳机
```

有例子在手，AI秒懂你要的是什么风格。

### 3.5 武器五：说清楚不要什么

有时候，说清楚"不要什么"跟"要什么"一样重要。

❌ 没说清楚：
- "帮我写一篇产品软文" → AI可能写得太硬广告
- "分析下这份报告" → AI可能给你整一堆废话

✅ 说了不要：
- "帮我写一篇产品软文，要求：不要硬广告感，要像朋友推荐一样自然；不要使用'卓越'、'极致'这类夸张词汇；控制在800字以内"
- "分析下这份报告，要求：只输出核心结论，不要铺垫废话；不要超过500字；用 bullet point 呈现关键发现"

### 3.6 武器六：指定输出格式

你想要什么格式，说清楚。

- "用表格呈现，列分别是：姓名、职位、入职时间、年薪"
- "用Markdown格式输出，包含H2标题、项目符号和表格"
- "用JSON格式输出，包含code、name、description三个字段"
- "按照以下结构输出：【问题】【分析】【建议】【行动计划】"

格式越明确，你拿到的东西越容易用。

### 3.7 武器七：分步骤提问

复杂问题不要一次问完，分成几步来。

❌ 一次问太多：
- "帮我分析下要不要从北京跳槽到上海，考虑工作机会、房价、孩子教育、生活成本、个人发展，给出具体建议" → AI容易顾此失彼

✅ 分步骤问：
```
第一步：帮我对比下北京和上海的互联网工作机会，重点看产品经理岗位的薪资和需求量
第二步：对比下两地的房价和租房成本，假设我在北京五环租一个一居室，上海内环的对应选择是什么
第三步：对比下两地的教育资源和落户政策
第四步：基于以上分析，给出我这个情况的跳槽建议
```

分步骤的好处是：AI每次处理的信息量更少，思考更深入；你在中间还可以根据回答调整后续问题。

## 四、框架篇：Structured Prompt（结构化提示词）

### 4.1 为什么需要框架？

你可能会想："每次都要想这么多要素，好累啊。"

没错，这就是为什么出现了各种"提示词框架"——帮你把该想的要素系统性地想一遍，不用每次从零开始。

框架不是必须的，但用框架能帮你：
1. 不遗漏重要信息
2. 写出的提示词更规范
3. 调试和优化更容易

下面给你介绍几个最常用的框架。

### 4.2 COSTAR框架——最全面的六边形战士

COSTAR是当前最流行的提示词框架之一，来自新加坡的一次GPT-4提示词竞赛冠军。它把提示词分成六个要素：

| 要素 | 含义 | 你要回答的问题 |
|------|------|----------------|
| **C**ontext | 上下文 | 我在什么背景/情况下？ |
| **O**bjective | 目标 | 我想要什么结果？ |
| **S**tyle | 风格 | 我想要什么感觉/调性？ |
| **T**one | 语气 | 我想要什么态度？ |
| **A**udience | 受众 | 文章给谁看？ |
| **R**esponse | 响应 | 我要什么格式的输出？ |

**举个例子：**

不用COSTAR（很泛）：
```
帮我写一篇关于AI的文章
```

用COSTAR：
```
# Context（上下文）
我们是一家面向职场新人的职业成长公众号，读者主要是工作1-3年的年轻人。

# Objective（目标）
写一篇介绍AI如何帮助职场人提升效率的文章，让读者了解3个具体可上手的AI工具使用方法。

# Style（风格）
参考36氪、虎嗅的商业科技报道风格：专业但不晦涩，有数据有案例。

# Tone（语气）
积极、鼓励、务实，像一个懂行的朋友在分享经验。

# Audience（受众）
工作1-3年的职场新人，对AI有兴趣但不知道怎么用，担心被AI取代。

# Response（响应）
格式：开篇hook（吸引眼球的问题或数据）+ 3个工具介绍（每个包含：工具名称、使用场景、具体示例）+ 结尾行动号召
长度：1500-2000字
```

看出来了吗？用了COSTAR之后，AI知道它是在帮一家公众号写给职场新人看的文章，风格要像商业科技媒体，语气要积极务实，格式和长度都有了明确要求。输出质量会高很多。

### 4.3 CRISPE框架——带角色扮演的专业版

CRISPE比COSTAR多了一个R（Role角色），适合需要AI扮演特定身份的场景：

| 要素 | 含义 |
|------|------|
| **C**apacity | 能力/角色 |
| **R**ole | 扮演什么身份 |
| **I**nsight | 背景洞察 |
| **S**tatement | 任务声明 |
| **P**ersonality | 人格特质 |
| **E**xperiment | 实验/探索 |

**举个例子（让AI扮演资深产品经理）：**

```
# Capacity/Role
你是一位拥有15年经验的产品专家，曾在字节跳动、阿里巴巴等公司负责过DAU过亿的产品。

# Insight
我们是一家做B2B SaaS的创业公司，目前100人，做项目管理工具。最近发现大客户流失严重，需要你帮忙分析问题。

# Statement
请分析大客户流失的可能原因，并给出优先级排序的建议解决方案。

# Personality
直接、务实、数据驱动。喜欢用框架和模型来思考问题。

# Experiment
在给出方案时，也请提供2-3个备选方案，并说明各自的优劣。
```

### 4.4 ICIO框架——快速简单任务用

如果COSTAR太复杂，试试这个精简版：

| 要素 | 含义 |
|------|------|
| **I**nstruction | 指令（要做什么） |
| **C**ontext | 上下文（背景信息） |
| **I**nput | 输入（要处理什么） |
| **O**utput | 输出（期望的格式） |

**适合场景**：翻译、润色、快速总结、简单问答

### 4.5 BROKE框架——目标驱动型

BROKE的特别之处是引入了OKR（目标与关键成果）的思想，适合需要明确目标和衡量标准的商业场景：

| 要素 | 含义 |
|------|------|
| **B**ackground | 背景 |
| **R**ole | 角色 |
| **O**bjective | 目标 |
| **K**ey Results | 关键成果 |
| **E**volve | 演化/迭代优化 |

**适合场景**：商业分析、项目规划、战略制定

## 五、进阶篇：让AI"思考"更清楚的技术

### 5.1 链式思考（Chain of Thought）——让AI展示推理过程

你有没有遇到过这种情况：问AI一个需要推理的问题，它直接给答案，结果答案还是错的？

链式思考（CoT）就是来解决这个问题的。它的核心思想是：**让AI先把推理过程说出来，再给答案**。

**方法很简单：加一句"请一步一步思考"**

❌ 直接要答案：
```
小明有12个苹果，送给小红一半，又买了8个。小明现在有多少个苹果？
答案：
```

✅ 加上CoT引导：
```
小明有12个苹果，送给小红一半，又买了8个。小明现在有多少个苹果？
请一步一步思考，展示你的计算过程。
```

加了这句引导后，AI会先写出推理过程：
1. 小明有12个苹果
2. 送给小红一半：12 ÷ 2 = 6（送出了6个）
3. 剩下：12 - 6 = 6（个）
4. 又买了8个
5. 现在有：6 + 8 = 14（个）

**为什么CoT有效？** 因为AI在展示推理的过程中，相当于自己检查了一遍逻辑。如果推理步骤有错，它可以及时发现和修正。

**进阶用法：Few-shot CoT**

如果你觉得单句引导不够，可以给AI展示几个完整的推理例子：

```
示例1：
问题：小张有5个橘子，给了弟弟3个，又收到了4个。小张现在有几个橘子？
推理：小张有5个橘子。给了弟弟3个，剩下5-3=2个。又收到4个，变成2+4=6个。
答案：6个

示例2：
问题：面包店上午卖了32个面包，下午比上午多卖了15个。一共卖了多少个？
```

### 5.2 思维树（Tree of Thoughts）——探索多种可能

有些问题不是只有一个正确答案，而是有多种可能的解决方案。这时候链式思考就不够用了，你需要"思维树"。

思维树的核心思想是：**在关键决策点，同时探索多个可能的方向，然后评估哪个方向最好**。

**举个例子：要不要转行**

```
问题：要不要从传统行业转到互联网？

【思维树探索】

方向A：转行做产品经理
├── 优势：门槛相对低，薪资高，成长快
├── 劣势：要从头开始，可能降薪入职
└── 需要考虑：我有没有产品思维？能不能接受初期压力？

方向B：转行做运营
├── 优势：门槛更低，需求大，可以逐步过渡
├── 劣势：薪资天花板可能比产品低
└── 需要考虑：我对运营有没有兴趣？

方向C：留在原行业，用AI赋能
├── 优势：不需要转行，利用AI提升效率
├── 劣势：可能错过互联网红利
└── 需要考虑：我现在的行业有没有数字化机会？

【评估】
从短期稳定性看：B > C > A
从长期发展看：A > B > C
从学习曲线看：C > B > A

【建议】
如果你35岁以下、风险承受能力较强，建议方向A（产品经理）
如果你求稳、希望平稳过渡，建议方向B（运营）
如果你对现在的行业有感情、只是想提升效率，建议方向C
```

**思维树适合什么场景？**
- 重大决策（职业、创业、投资）
- 需要多种解决方案的创意工作
- 需要权衡利弊的分析报告

### 5.3 ReAct——边想边做，让AI使用工具

ReAct（Reasoning + Acting）是更高级的技术，它让AI不仅能思考，还能调用外部工具来完成任务。

典型的ReAct循环是这样的：
```
Thought（思考）→ Action（行动）→ Observation（观察）→ Thought → Action → ...
```

**举个例子：研究一个你不了解的主题**

```
你是一个研究助手，可以通过搜索来回答问题。

问题：特斯拉FSD自动驾驶技术和华为ADS2.0相比，哪个更厉害？

Thought 1：我需要先了解特斯拉FSD的最新进展
Action 1：搜索"特斯拉FSD 2025最新进展"
Observation 1：[搜索结果显示FSD V13版本已实现端到端架构...]

Thought 2：现在我需要了解华为ADS2.0的技术特点
Action 2：搜索"华为ADS2.0技术特点 2025"
Observation 2：[搜索结果显示华为ADS采用激光雷达+视觉融合方案...]

Thought 3：我现在有足够信息进行对比分析了
Action 3：整理对比分析报告
...
```

**ReAct适合什么场景？**
- 需要实时信息的问答（新闻、股价、天气）
- 多跳推理的复杂问题
- 需要验证事实的研究任务

### 5.4 自洽性（Self-Consistency）——让AI"投票"决定答案

你有没有发现，同一个问题问AI几遍，有时候会得到不同的答案？Self-Consistency就是利用这个特点来提高准确率的。

**原理是这样的：**
1. 让AI用不同的方式思考同一个问题（比如用方法A、方法B、方法C）
2. 收集每种方法得出的答案
3. 看哪个答案出现次数最多，就选那个

**为什么有效？**
因为正确答案通常只有一种，而错误答案可能有无数种。如果某种"错误思路"出现了好几次，它也会被计入投票。所以通过多方法验证，正确的答案往往能脱颖而出。

**Self-Consistency适合什么场景？**
- 重要的推理任务（数学题、逻辑分析）
- 需要高准确率的场景
- 可以接受较高成本（多次调用）

## 七、避坑篇：新手最容易犯的错误

### 7.1 问题太宽泛

❌ "帮我写篇文章" → 不知道写什么、给谁看、多长
✅ "帮我写一篇面向00后大学生的618购物节种草文案，200字以内，要有网感"

### 7.2 不给上下文

❌ "分析下这个方案有什么问题" → 不知道方案是什么、背景是什么
✅ "这是我们公司的用户增长方案，目标用户是一线城市白领，目前预算50万，请分析下有什么问题"

### 7.3 期望AI完美

AI不是万能的，它会出错、会有幻觉、会有理解偏差。你要：
- 对AI的输出保持批判性思维
- 重要内容要核实
- 把AI当作助手，不是替你做决定的神

### 7.4 一次问太多

❌ 把一整个项目的要求一次说完 → 信息过载，AI顾此失彼
✅ 分步骤进行，每个步骤拿到满意结果再推进下一步

### 7.5 不会追问

拿到AI的回答不满意？别急着换工具或放弃，追问才是王道：
- "你说的第二点能展开说说吗？"
- "能给我一个具体的例子吗？"
- "这个方案有什么风险？"
- "如果我预算不够，有没有办法简化？"

好的提示词工程师，都是通过追问一步步把AI的输出打磨成自己想要的样子。

### 7.6 不会复用

找到一个好的提示词？保存下来！
- 可以为它取个名字
- 记录它适合什么场景
- 记录你后来做了哪些调整

你积累的提示词模板库，才是真正属于你的AI使用能力。

## 八、资源篇：继续学习的方向

### 8.1 推荐学习路径

**第一阶段：会用（1-2天）**
- 理解什么是提示词
- 学会7种基础技巧
- 能处理日常简单任务

**第二阶段：用好（1周）**
- 学会1-2个提示词框架
- 能处理中等复杂任务
- 能通过追问优化输出

**第三阶段：精通（长期）**
- 理解各种进阶技术（CoT、ToT、ReAct）
- 能针对不同场景设计提示词
- 能评估和优化提示词

### 8.2 进阶学习方向

- **CoT（链式思考）**：让AI展示推理过程，提高复杂任务的准确率
- **ToT（思维树）**：探索多种可能，适合重大决策
- **ReAct**：AI调用工具，适合需要实时信息的研究任务
- **Self-Consistency**：多路径验证，适合高准确率要求的任务
- **Agentic Prompting**：让AI自主规划和执行任务

### 8.3 实践建议

1. **每天用**：学再多不用等于没学。找一件你每天要做的事，试试用AI帮忙
2. **保存模板**：好用的提示词保存下来，以后可以复用
3. **分享交流**：跟朋友交流AI使用心得，互相学习
4. **关注更新**：AI工具和提示词技术都在快速迭代，保持学习

## 九、总结

最后总结一下这篇长文的核心要点：

1. **提示词是你跟AI的沟通方式**：说清楚、说具体、说完整
2. **AI是根据上下文生成输出的**：给的信息越多，结果越准
3. **七种基础武器**：明确需求、说清身份、给AI角色、举例子、说不要什么、定格式、分步骤
4. **框架帮你系统思考**：COSTAR、CRISPE、ICIO、BROKE各有适用场景
5. **进阶技术提升质量**：CoT、ToT、ReAct、Self-Consistency
6. **多练多用才是王道**：光学不练假把式

记住，提示词工程不是玄学，是可以学习、可以练习、可以提升的技能。你不需要成为专家，只需要比大多数人更会提问，就能让AI成为你的超级助手。
