# 智能体搭建完全指南：dify / n8n / coze 三大平台深度解析

> [!NOTE] 写给谁看的？
> 这是一篇面向小白的入门教程，同时也是有经验的开发者可以收藏的速查手册。不管你是刚听说"AI Agent"这个概念，还是已经在玩各种Bot的进阶玩家，这篇文章都能给你带来价值。

## 先搞清楚几个核心概念

### 什么是 AI Agent（智能体）？

用大白话来说，AI Agent就是一个**会思考、会做事的AI程序**。它不是那种你问一句它答一句的简单机器人，而是一个能帮你完成复杂任务的"数字员工"。

打个比方：

- **普通AI** 就像是个只会照本宣科的实习生，你说干啥就干啥，不会变通
- **AI Agent** 就像是个有经验的老员工，你给个目标，它自己想办法搞定

举几个实际例子：

| 任务 | 普通AI | AI Agent |
|------|--------|----------|
| "帮我查天气" | 直接告诉你天气 | 先查天气，再问你"要不要根据天气推荐穿衣" |
| "帮我写周报" | 随便写一篇 | 先问你本周做了什么，再从邮件/日历里收集信息，再生成个性化周报 |
| "帮我分析竞品" | 瞎编一通 | 先去网上搜真实数据，再分析，再给你结论 |

### 智能体能干什么？

说起来能干的太多了，我给你列几个最常见的场景：

**1. 智能客服**
不用你手动回复，AI自动回答客户问题，能聊天的同时还能查订单、推荐产品、转人工。

**2. 内容创作助手**
写文章、起标题、做翻译、润色文案，扔给它素材，它给你成品。

**3. 数据分析助手**
扔给它一堆数据，让它帮你分析趋势、找规律、写报告。

**4. 个人助理**
帮你管日程、发邮件、记笔记、设置提醒，一句话的事儿。

**5. 业务流程自动化**
跨系统操作、数据同步、报表生成，省去大量重复劳动。

### 三大核心能力

一个能打的智能体，通常具备这三个能力：

**1. 理解能力**
能听懂人话，不只是匹配关键词，而是真正理解你的意思。

**2. 规划能力**
能把大任务拆成小步骤，知道先干啥后干啥。

**3. 执行能力**
能调用各种工具，查资料、发邮件、调API，不只是在那干巴巴地回答问题。

## 一、平台概览：选哪个平台？

好，现在你大概知道智能体是啥了。接下来最重要的问题是：**我用哪个平台来搭建？**

目前最火的三个平台是 **Dify**、**n8n**、**Coze（扣子）**。它们各有特点：

### 快速对比

| 维度 | Dify | n8n | Coze（扣子） |
|------|------|------|--------------|
| 定位 | AI应用开发平台 | 工作流自动化 + AI | 对话式AI Bot平台 |
| 上手难度 | 中等 | 较难 | 简单 |
| 是否开源 | 完全开源 | Fair-code开源 | 部分开源 |
| 部署方式 | 本地/云端 | 本地/云端 | 纯云端 |
| AI能力深度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| 自动化能力 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 易用性 | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐⭐ |

### 我的选型建议

**选 Coze 的情况：**
- 纯新手，只想快速体验
- 不懂技术，不想折腾代码
- 想快速做个客服Bot发布到微信/飞书
- 想用别人做好的Bot（平台有商店）

**选 Dify 的情况：**
- 想认真学技术，以后自己搭复杂系统
- 企业需要私有化部署，数据要自主
- 需要做RAG（知识库问答）场景
- 想把AI能力集成到自己产品里

**选 n8n 的情况：**
- 主要想做各种办公自动化
- 需要连接很多外部服务（Gmail、Slack、Notion...）
- 喜欢拖拖拽拽搭流程
- 有一定技术基础

> [!TIP] 新手建议
> 如果你完全不懂AI，推荐先用 Coze 体验一下，10分钟就能跑起来。觉得不够用了再迁移到 Dify。

## 二、Dify — 最专业的AI应用开发平台

### Dify 是啥？

Dify（名字来自"Do It For You"）是一个开源的AI应用开发平台。它的定位是**让开发者能快速构建、部署、管理基于大语言模型的应用**。

你可以把它理解为一个"AI应用的脚手架"，它帮你处理了：
- Prompt工程
- 上下文管理
- 对话状态
- 多轮对话
- RAG（知识库检索）
- API发布

你自己就不用从零写这些底层代码了。

### 核心特点

**1. DAG可视化工作流**
用拖拽的方式搭业务流程，节点之间的数据流向一目了然。

**2. 原生支持RAG**
内置知识库系统，上传文档就能做问答，不用额外集成。

**3. Agent智能体**
支持ReAct、Function Calling等高级Agent能力。

**4. API优先**
一键生成API，把AI能力集成到任何地方。

### 适用场景

- 企业内部知识库问答系统
- 复杂AI应用快速原型
- 需要私有化部署的场景
- 多模型协作的复杂系统

### Docker部署 Dify（保姆级教程）

下面手把手教你在本地电脑上装Dify。

**第一步：装Docker**

Docker是容器技术，简单理解就是"把软件打包成一个可以到处跑的盒子"。

1. 去 https://www.docker.com 下载Docker Desktop
2. 安装并启动（Mac可能需要给权限）
3. 打开终端，输入 `docker --version`，看到版本号说明装好了

**第二步：克隆代码并启动**

```bash
# 克隆Dify代码
git clone https://github.com/langgenius/dify.git

# 进入docker目录
cd dify/docker

# 复制环境配置
cp .env.example .env

# 启动所有服务
docker compose up -d

# 检查服务状态
docker compose ps
```

**第三步：初始化**

1. 打开浏览器，访问 http://localhost:80
2. 第一次访问会让你创建管理员账号
3. 登录后就能开始用了

> [!IMPORTANT] 常见问题
> - **Mac上启动慢**：正常，Docker第一次启动要下载镜像，耐心等几分钟
> - **端口80被占用**：修改.env文件，把`EXPOSE_NGINX_PORT`改成其他端口比如8080
> - **内存不够**：Dify比较吃内存，建议电脑有8GB以上内存

## 三、n8n — 最强大的工作流自动化工具

### n8n 是啥？

n8n（发音"n-eight-n"）是一个开源的工作流自动化工具。它的定位是**连接一切、自动化一切**。

如果说Dify是专门为AI打造的，那n8n就是通用自动化平台。只不过现在AI火起来了，它也加了AI能力进去。

### 核心特点

**1. 800+集成**
能连接各种服务：Gmail、Slack、Notion、Stripe、Airtable、GitHub...基本上你用到的它都能接。

**2. 纯可视化拖拽**
不用写代码，拖拖拽拽就能搭流程。

**3. 代码可嵌入**
支持JavaScript和Python，想怎么扩展就怎么扩展。

**4. 自托管**
完全开源，可以部署在自己服务器上，数据完全自主。

### 适用场景

- 跨系统数据同步（比如CRM和ERP之间的数据流转）
- 办公自动化（自动发邮件、整理文件、生成报表）
- AI辅助的数据处理
- 需要深度定制的企业自动化场景

### Docker部署 n8n（保姆级教程）

**方式一：快速尝鲜（单容器）**

```bash
# 一行命令搞定
docker run -d --name n8n \
  -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n:latest
```

然后打开 http://localhost:5678 就能用了。

**方式二：生产环境（推荐，推荐配PostgreSQL）**

创建 `docker-compose.yml`：

```yaml
version: '3'
services:
  n8n:
    image: n8nio/n8n:latest
    restart: always
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=你的密码
      - DB_TYPE=postgresdb
      - DB_POSTGRESDB_HOST=n8n-db
      - DB_POSTGRESDB_PORT=5432
      - DB_POSTGRESDB_DATABASE=n8n
      - DB_POSTGRESDB_USER=n8n
      - DB_POSTGRESDB_PASSWORD=n8n密码
    volumes:
      - n8n_data:/home/node/.n8n
    depends_on:
      - n8n-db
  
  n8n-db:
    image: postgres:15
    environment:
      - POSTGRES_DB=n8n
      - POSTGRES_USER=n8n
      - POSTGRES_PASSWORD=n8n密码
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  n8n_data:
  postgres_data:
```

运行：

```bash
docker compose up -d
```

## 四、Coze（扣子）— 最简单的Bot开发平台

### Coze 是啥？

Coze（扣子）是字节跳动出的AI Bot开发平台。它的定位是**让任何人都能快速创建AI Bot**。

一句话：**不用写代码，拖拖拽拽就能做Bot**。

### 核心特点

**1. 零基础友好**
不需要懂技术，会用电脑就能玩。

**2. Bot商店**
有现成的Bot可以直接用，不用从零开始。

**3. 多平台发布**
一键发布到飞书、微信、抖音、Discord、Telegram...全平台覆盖。

**4. 插件生态**
有丰富的插件可以用，查天气、搜新闻、调API，都能搞定。

### 适用场景

- 快速验证AI产品想法
- 做客服机器人
- 社交媒体Bot
- 个人助手

### 使用 Coze（云端，无需部署）

Coze是国内版（coze.cn）和国际版（coze.com），都是云端服务，不用安装：

1. 打开 https://www.coze.cn（国内版）
2. 注册/登录
3. 点击"创建Bot"
4. 拖拖拽拽配置
5. 发布

完事！

## 五、三平台核心功能对比

### 功能矩阵

| 功能维度 | Dify | n8n | Coze |
|---------|------|------|------|
| **许可证** | Apache 2.0 | Fair-Code | Apache 2.0 |
| **自托管** | ✅ 支持 | ✅ 支持 | ❌ 仅云端 |
| **可视化工作流** | ✅ DAG引擎 | ✅ 节点图 | ✅ 流程画布 |
| **RAG知识库** | ✅ 原生支持 | ✅ 需配置 | ✅ 原生支持 |
| **AI Agent** | ✅ 原生支持 | ✅ 插件支持 | ✅ 原生支持 |
| **多模型支持** | ✅ 主流全覆盖 | ✅ 灵活接入 | ⚠️ 受限（国内版） |
| **API发布** | ✅ 一键生成 | ⚠️ 需配置 | ⚠️ 受限 |
| **集成数量** | 有限 | 800+ | 400-600+ |

### 工作流引擎对比

**Dify 工作流**

Dify用的是DAG（有向无环图）架构。这是它区别于其他平台的核心优势：

- **确定性执行**：每个节点的输出可预测，好调试
- **并行处理**：支持节点并行，效率高
- **循环支持**：有迭代节点，可以处理循环逻辑
- **全链路可观测**：每个节点的输入输出、token消耗都能看到

核心节点：
- LLM节点：调用大语言模型
- 知识检索节点：配置RAG检索策略
- 代码执行节点：Python/JavaScript
- HTTP请求节点：调外部API
- 条件分支节点：if-else逻辑
- 模板转换节点：格式化输出

**n8n 工作流**

n8n用的是节点图模式，像搭积木一样：

- 触发器节点：Webhook、定时器、事件
- AI节点：LLM、Prompt、Memory
- 工具节点：HTTP请求、代码、计算器
- 集成节点：800+预置连接器

**Coze 工作流**

Coze的画布式设计，强调拖拽即用：

- 触发器节点：手动、定时、事件
- LLM节点：调用大语言模型
- 插件节点：访问外部服务
- 知识节点：RAG检索
- 条件节点：分支逻辑
- 循环节点：重复执行

### RAG知识库能力

**Dify** 的RAG最专业：
- 支持PDF、Word、TXT、Markdown、HTML、CSV
- 自动分块（典型500-1000 tokens）
- 多种Embedding模型可选
- 支持向量数据库：pgvector、Qdrant、Weaviate、Milvus、Pinecone
- 支持混合检索、rerank

**Coze** 的RAG面向对话优化：
- 导入格式支持Excel、PDF、TXT、Word
- 可结合搜索插件保证时效性
- 知识库节点可直接嵌入工作流

**n8n** 的RAG需要配置：
- 通过节点组合构建RAG管道
- 灵活性高但需要一定技术基础
- 支持Pinecone、Qdrant、Weaviate

## 七、三平台实战案例

### 案例1：企业知识库问答系统

**推荐平台：Dify**

**需求：** 公司有大量内部文档（新员工手册、产品文档、流程规范），需要做个问答系统，员工问问题，AI自动从文档里找答案。

**实现步骤：**

1. **上传文档到知识库**
   - 在Dify里创建知识库
   - 上传PDF、Word等格式的文档
   - 系统自动分块和向量化

2. **创建RAG应用**
   - 新建一个"助手"类型的应用
   - 把知识库关联进去
   - 编写系统提示词，设定AI的角色和行为

3. **配置检索策略**
   - 开启混合检索（语义+关键词）
   - 设置top_k=5，score_threshold=0.7
   - 开启rerank提升精度

4. **发布并接入**
   - 一键发布为API
   - 嵌入公司内部系统（钉钉、企业微信等）

**效果：** 员工有问题直接问，秒级响应，准确率85%+。

### 案例2：跨系统数据自动化

**推荐平台：n8n**

**需求：** 每天早上把昨天CRM里的新订单同步到ERP系统，同步完成后发钉钉通知给相关人员。

**实现步骤：**

1. **创建定时触发器**
   - 设置每天早上8点自动执行

2. **连接CRM读取数据**
   - 用n8n的Salesforce节点
   - 查询昨天的新订单

3. **数据清洗转换**
   - 用代码节点处理数据格式
   - 适配ERP系统的字段要求

4. **写入ERP系统**
   - 用SAP/金蝶等ERP的API节点
   - 批量创建订单记录

5. **发送通知**
   - 用钉钉节点
   - 发送同步结果通知

**效果：** 原来每天手动处理2小时的工作，变成全自动。

### 案例3：智能客服机器人

**推荐平台：Coze**

**需求：** 做个微信客服Bot，能自动回答客户关于产品、售后、订单的问题。

**实现步骤：**

1. **创建Bot**
   - 在Coze里新建一个Bot
   - 设定角色为"专业客服"

2. **配置知识库**
   - 上传产品FAQ、常见问题等文档
   - AI会自动从知识库里找答案

3. **添加插件**
   - 接入订单查询插件
   - 接入物流查询插件

4. **设计工作流**
   - 意图识别 → 分流 → 处理 → 回复
   - 复杂问题自动转人工

5. **发布到微信**
   - 企业微信或公众号接入
   - 对接Bot API

**效果：** 7x24小时服务，40%+的常见问题自动处理。

## 八、最佳实践与避坑指南

### 新手常见问题

**问题1：选错平台**
很多新手上来就选最复杂的，结果被劝退了。

建议：**先用Coze体验，确定需求后再选Dify或n8n**。

**问题2：Prompt写得不好**
觉得AI不够"智能"，其实是Prompt没写好。

建议：Prompt要具体、有例子、设边界。参考这篇教程的Prompt部分。

**问题3：知识库质量差**
问答效果不好，根源往往是知识库文档质量差。

建议：**文档要结构清晰、内容准确、定期更新**。

**问题4：忽略成本**
AI调用是要花钱的，没控制好月底账单吓人。

建议：**设置调用限额，监控Token消耗，用便宜模型处理简单任务**。

### 进阶技巧

**技巧1：多模型组合**
- 简单任务用GPT-4o-mini，便宜又快
- 复杂任务用GPT-4o或Claude
- 国产场景用DeepSeek，省钱

**技巧2：缓存为王**
- 相同问题缓存答案，避免重复调用
- 静态数据定期缓存，减少API调用

**技巧3：降级方案**
- AI服务不可用时，有备用方案
- 比如查不到答案时返回"请稍后再试"
