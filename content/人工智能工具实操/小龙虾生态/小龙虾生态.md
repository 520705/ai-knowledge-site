## 一、项目概述

### 1.1 什么是「小龙虾生态」

「小龙虾生态」并非生物学概念，而是指以 **OpenClaw** 为核心的开源 AI Agent 框架生态系统。OpenClaw 项目以小龙虾（lobster）为标志和代号，其命名来源于项目发起人 Mario Zechner 对海洋生物的偏爱。在中文社区中，「小龙虾」已逐渐成为 OpenClaw 的非官方昵称，形成了独特的「小龙虾生态」——涵盖核心框架、社区插件、技能市场（ClawHub）以及与之竞争或互补的项目如 **Hermes Agent**。

截至2026年4月，OpenClaw 在 GitHub 上已累计获得超过 **35.9万颗星标**，成为开源 AI Agent 领域最具影响力的项目之一。其生态系统包括：

- **OpenClaw**：多渠道个人助理运行时
- **Pi Framework**：底层极简 Agent 框架
- **ClawHub**：技能市场（3,200+ 技能插件）
- **Hermes Agent**：自进化智能体引擎（Nous Research 开发）

### 1.2 OpenClaw 核心定位

OpenClaw 是一个**自托管的个人 AI 助手运行时**，其核心理念是：

> *"Any OS. Any Platform. The lobster way."*

它能够将你的 AI 助手连接到各种即时通讯平台（Discord、Telegram、WhatsApp、Slack、Signal 等），让你可以通过任意聊天应用与 AI 助手交互，同时保持对数据和隐私的完全控制。

**核心特点**：

- **自托管**：数据存储在本地，不依赖云服务
- **多渠道**：支持 20+ 聊天平台的统一接入
- **工具调用**：支持代码执行、文件操作、网页浏览等
- **记忆系统**：跨会话持久化上下文
- **多 Agent 路由**：支持隔离的工作区和会话管理
- **MIT 开源协议**：代码完全开放可审计

### 1.3 Hermes Agent 核心定位

Hermes Agent 是由 **Nous Research** 开发的新一代自进化 AI Agent，于 **2026年2月正式发布**。与 OpenClaw 的「广度优先」策略不同，Hermes 采用「深度优先」策略，专注于让 AI Agent 越用越聪明。

**核心特点**：

- **持久记忆**：基于 SQLite + FTS5 的跨会话记忆系统
- **自进化技能**：AI 自动从解决问题中生成可复用技能
- **多渠道网关**：支持 Telegram、Discord、Slack、WhatsApp、Signal
- **40+ 内置技能**：开箱即用的工具集
- **MLOps 能力**：支持批量处理、强化学习训练、轨迹导出
- **MIT 开源协议**

## 二、技术架构深度解析

### 2.1 OpenClaw 架构：Gateway 模式

OpenClaw 采用 **Gateway 网关架构**，整体结构分为以下几个层次：

```
┌─────────────────────────────────────────────────────────────┐
│                      Chat Platforms                          │
│  (Discord, Telegram, Slack, WhatsApp, Signal, etc.)        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    OpenClaw Gateway                          │
│  - WebSocket Control Plane (port 18789)                     │
│  - Hub-and-Spoke Message Routing                           │
│  - Session & Memory Management                             │
│  - Channel Adapter Pattern                                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      Pi Agent SDK                           │
│  - Embedded in OpenClaw process                            │
│  - 4 Core Tools: read, write, edit, bash                    │
│  - System Prompt < 1000 tokens                             │
│  - Multi-Model Support                                     │
└─────────────────────────────────────────────────────────────┘
```

**Gateway 核心组件**：

| 组件 | 功能描述 |
|------|----------|
| **Channel Plugin Interface** | 适配器模式，统一处理各平台差异 |
| **Session Manager** | 持久化会话状态，支持分支回溯 |
| **Memory Manager** | 基于 Markdown 文件的静态记忆系统 |
| **Tool Executor** | Docker 沙箱执行危险操作 |
| **Multi-Model Router** | 支持 Anthropic、OpenAI、Google、xAI、Groq 等 |

### 2.2 Pi Framework：极简 Agent 框架

Pi 是 OpenClaw 的底层框架，由 Mario Zechner（libGDX 创始人）开发。它证明了**极简设计**可以与复杂的 Agent 框架相媲美。

**Pi 核心设计哲学**：

- **最小工具集**：仅 4 个核心工具（read、write、edit、bash）
- **紧凑系统提示**：控制在 1000 tokens 以内
- **4 种运行时模式**：交互式、JSON 输出、RPC、SDK 嵌入

**Pi 分层架构**：

| 层级 | 职责 |
|------|------|
| **pi-ai** | 统一 LLM API，支持多提供商 |
| **pi-agent-core** | Agent 循环核心逻辑 |
| **pi-coding-agent** | 编程专用增强层 |
| **pi-tui / pi-web-ui** | 用户界面层 |

### 2.3 Hermes Agent 架构：Engine 模式

Hermes Agent 采用 **Engine 自进化引擎架构**，其核心是一个**闭环学习系统**。

```
┌─────────────────────────────────────────────────────────────┐
│                     Entry Points                             │
│     (CLI, Gateway, ACP Python Library)                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    AIAgent (run_agent.py)                   │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────┐ │
│  │  Prompt Builder │  │ Provider        │  │ Tool        │ │
│  │  (压缩 + 缓存)  │  │ Resolution      │  │ Dispatch    │ │
│  └─────────────────┘  └─────────────────┘  └─────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    Tool Registry                            │
│     47 Tools organized in 19 Toolsets                       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  SQLite + FTS5 Memory                       │
│     持久化记忆 · 全局搜索 · 跨会话学习                        │
└─────────────────────────────────────────────────────────────┘
```

**核心组件详解**：

1. **Prompt Builder**：动态组装系统提示，支持压缩和缓存
2. **Provider Resolution**：支持多种 API 调用模式（chat completion、codex response、Anthropic mode）
3. **Tool Dispatch**：47 个工具组织成 19 个工具集
4. **Memory System**：SQLite + FTS5 全文本搜索，支持 LLM 摘要压缩

### 2.4 架构对比总结

| 维度 | OpenClaw (Gateway) | Hermes Agent (Engine) |
|------|-------------------|----------------------|
| **架构模式** | 网关调度型 | 自进化引擎型 |
| **工具数量** | 4 核心 + 扩展 | 47 内置 + 自定义 |
| **记忆系统** | Markdown 文件 | SQLite + FTS5 |
| **学习方式** | 人工编写技能 | AI 自动生成技能 |
| **设计哲学** | 广度优先 | 深度优先 |
| **适用场景** | 多渠道接入 | 长期进化 |

## 三、部署指南

### 3.1 OpenClaw 部署方案

#### 方案一：npm 全局安装（推荐个人用户）

**前置要求**：

- Node.js 24+（推荐）或 Node.js 22 LTS (22.14+)
- 2GB+ RAM
- 3-5GB 磁盘空间

**安装步骤**：

```bash
# 全局安装
npm install -g openclaw@latest

# 启动引导向导（同时安装守护进程）
openclaw onboard --install-daemon

# 打开控制面板
openclaw dashboard
```

访问 `http://127.0.0.1:18789/` 并输入生成的 token 即可开始使用。

#### 方案二：Docker 部署（推荐服务器环境）

**前置要求**：

- Docker Engine v24+（v27+ 推荐）
- Docker Compose v2.20+（v2.30+ 推荐）
- 2GB RAM（构建）/ 1GB RAM（运行）/ 4GB 推荐

**快速部署**：

```bash
# 克隆仓库
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# 运行官方安装脚本（自动构建或拉取镜像）
./docker-setup.sh

# 或使用预构建镜像（跳过 5-15 分钟构建时间）
export OPENCLAW_IMAGE="ghcr.io/openclaw/openclaw:latest"
./docker-setup.sh
```

**docker-compose.yml 示例**：

```yaml
version: '3.8'
services:
  openclaw:
    image: ghcr.io/openclaw/openclaw:latest
    container_name: openclaw
    restart: unless-stopped
    ports:
      - "18789:18789"
    volumes:
      - ~/.openclaw:/root/.openclaw
      - openclaw-workspace:/root/.openclaw/workspace
    environment:
      - OPENCLAW_API_KEY=${OPENCLAW_API_KEY}
    security_opt:
      - no-new-privileges:true
    read_only: true

volumes:
  openclaw-workspace:
```

**数据持久化**：

```bash
# 重要：配置和数据存储在 ~/.openclaw/
# 备份命令
tar -czf openclaw-backup-$(date +%Y%m%d).tgz ~/.openclaw
```

#### 方案三：Android/Termux 部署

```bash
# Termux 中安装
pkg update && pkg install nodejs
npm install -g openclaw@latest
openclaw onboard
```

```yaml
      test: ["CMD", "curl", "-f", "http://localhost:18792/health"]

```

```
├── skill-name/

│   └── resources/        # 资源文件
```

**SKILL.md 示例**：

```markdown
# GitHub Issue Creator

## Description
Automatically creates GitHub issues from natural language descriptions.

## Triggers
- "create issue"
- "file a bug"
- "report problem"

## Actions
1. Parse issue details from user input
2. Call GitHub API to create issue
3. Return issue URL

## Requirements
- GITHUB_TOKEN environment variable
- Repository write access

## Usage
Simply describe the issue you want to create.
```

### 4.2 Hermes Agent 自进化技能系统

```
┌─────────────────────────────────────────────────────────────┐

│                                                             │
│   ┌──────────┐    遇到复杂问题    ┌──────────┐              │
│   │  执行任务 │ ──────────────→ │ 生成技能 │              │

│        │                              ↓                     │
```

```bash

# 每周一 9 点执行代码审查
hermes schedule add "0 9 * * 1" --task weekly-review --platform discord

# 查看调度任务
hermes schedule list

# 删除调度任务
hermes schedule remove daily-report
```

**并行子 Agent**：

```bash
# 启动并行工作流
hermes spawn --count 3 --task research

```

## 五、生态对比分析

### 5.1 与 Dify 对比

| 维度 | OpenClaw | Dify |
|------|----------|------|
| **定位** | 个人 AI 助理运行时 | 低代码 LLM 应用平台 |
| **记忆系统** | Markdown 静态记忆 | RAG 向量检索 |
| **多 Agent** | 原生支持 | 有限支持 |
| **易用性** | 中等（需一定技术背景） | 低代码可视化 |
| **渠道接入** | 20+ 聊天平台 | Web/API 为主 |
| **部署方式** | 自托管 | 自托管/SaaS |
| **数据隐私** | 完全本地 | 部分云端选项 |
| **选型建议** | 需要 24/7 运行和深度定制的个人助手 | 快速构建 RAG 应用和原型 |

**核心差异**：
Dify 适合快速搭建知识库问答和简单 Agent 应用，10 分钟即可上线。但对于需要跨渠道、多 Agent 协作的场景，OpenClaw 更胜一筹。

### 5.2 与 Coze 对比

| 维度 | OpenClaw | Coze |
|------|----------|------|
| **开发方** | 开源社区 | 字节跳动 |
| **数据安全** | 完全本地 | 云端（数据由字节跳动托管） |
| **成本** | 仅 API 费用 | 免费但数据不自主 |
| **定制性** | 高（代码级别） | 中等（插件市场） |
| **部署** | 自托管 | 仅云端 |
| **多 Agent** | 原生 | 有限 |
| **选型建议** | 企业级、高安全需求 | 预算有限、非技术人员 |

**业界评价**：

## 八、安全考量

### 8.1 OpenClaw 安全实践

**安全配置示例**：

```json5
{
  // WhatsApp 白名单
  channels: {
    whatsapp: {
      allowFrom: ["+86138*******"],
      groups: {
        "*": {
          requireMention: true
        }
      }
    }
  },
  
  // 消息安全
  messages: {
    groupChat: {
      mentionPatterns: ["@openclaw"]
    }
  },
  
  // 工具执行策略
  tools: {
    execPolicy: "allowlist",
    allowedTools: ["read", "write", "bash"],
    deniedTools: ["rm", "format"]
  }
}
```

**风险提示**：

- ClawHub 约 7.1% 的技能可能存在凭据泄露风险
- 建议使用 VirusTotal 扫描第三方技能
- 生产环境建议启用 `execPolicy`

### 8.2 Hermes Agent 安全实践

**五层防御体系**：

1. **授权层**：API Key 和 OAuth 验证
2. **审批层**：高危操作需要确认
3. **隔离层**：Docker 沙箱执行
4. **过滤层**：输入输出扫描
5. **注入扫描**：防止 prompt injection

**容器加固建议**：

```yaml
security_opt:
  - no-new-privileges:true
read_only: true
cap_drop:
  - ALL
pids_limit: 100
```
