# 小龙虾生态完全入门指南

> 如果你看到"小龙虾"三个字一脸懵逼，这篇文章就是给你写的。我会用人话解释这是什么、有什么用、怎么一步步用起来。

---

## 先搞清楚：小龙虾生态到底是什么？

### 这不是教你养小龙虾

「小龙虾生态」其实是 AI 圈子里的一套**开源工具组合**，因为项目 logo 是只小龙虾（英文叫 lobster），所以中文社区就叫它「小龙虾」。

这套工具能帮你做一个**属于你自己的 AI 助手**，而且是完全跑在你自己的电脑/服务器上的——不用把数据交给 ChatGPT 之类的第三方。

简单说就是：

> **OpenClaw = 你自己部署的私人版 ChatGPT，还能接微信/Telegram/Discord**

### 小龙虾生态里有什么？

| 项目 | 定位 | 类比 |
|------|------|------|
| **OpenClaw** | 核心框架，搞定多平台接入 | 像一个万能插座 |
| **Pi Framework** | 底层技术，小而美 | 插座的内部电路 |
| **Hermes Agent** | 另一个框架，主打"越用越聪明" | 另一种设计的插座 |
| **ClawHub** | 插件市场，装完就能用 | 插座的各种功能扩展 |

## 小龙虾生态能干什么？

### 场景一：24小时在线的客服

你有个 Discord 服务器或者 Telegram 群，想弄个 AI 机器人自动回答问题？

用 OpenClaw，5 分钟就能接上一个会聊天的 AI——而且它能记住之前聊过什么。

### 场景二：个人助手

"帮我查下今天天气"、"提醒我下午三点开会"、"帮我写封邮件"——这些喊一嗓子就能搞定，不用打开一堆 App。

### 场景三：自动化工作流

每天自动整理笔记、自动发日报、自动备份文件……这些重复性工作交给 AI 做，省下的时间摸鱼不香吗？

## 小龙虾 vs Hermes：该选哪个？

这是新人最常问的问题。简单粗暴地说：

| 如果你是… | 推荐用… | 原因 |
|-----------|---------|------|
| 想要快速搞定多平台接入 | **OpenClaw** | 支持 20+ 平台，装上就能用 |
| 想让 AI 越用越懂你 | **Hermes** | 自动学习，从聊天中进化 |
| 纯小白第一次玩 | **OpenClaw** | 社区大、教程多、出问题好找人问 |
| 搞机器学习/RLHF | **Hermes** | 自带训练数据收集功能 |

**我的建议**：先玩 OpenClaw，上手了再试试 Hermes。

## 安装前需要准备什么？

### 硬件要求（很低！）

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|
| 内存 | 2GB | 4GB+ |
| 硬盘 | 10GB | 20GB+ SSD |
| CPU | 任意双核 | 任意四核 |

> 其实树莓派都能跑，就是会慢一点。

### 软件要求

| 软件 | 用途 | 备注 |
|------|------|------|
| **Node.js 22+** | 运行 OpenClaw | 必装 |
| **Docker** | 容器化部署（推荐） | 不会也可以用 npm |
| **Git** | 下载代码 | 可选 |
| **API Key** | 调用 AI 模型 | 最关键，准备好钱袋子 |

### 关于 API Key 的钱的问题

OpenClaw 本身是免费的，但调用 GPT、Claude 这些 AI 是要花钱的。

**好消息是**：这些 API 按量计费，个人轻度使用一个月可能就几美元。

**推荐的 API 提供商**：

| 提供商 | 模型 | 价格 | 适合场景 |
|--------|------|------|----------|
| **Anthropic (Claude)** | Claude 3.5 Sonnet | ~$3/百万token | 性价比高，推荐 |
| **OpenAI (GPT-4)** | GPT-4o | ~$5/百万token | 生态成熟 |
| **Groq** | Llama 3 | **免费** | 体验/测试用 |
| **Kimi** | moonshot | 国内可用 | 中文友好 |

> 注册地址：Anthropic 官网 / OpenAI 官网 / Groq 官网 / Kimi 官网

## 开始安装！手把手教程

### 方式一：Docker 安装（推荐，5分钟搞定）

Docker 就像一个"隔离的盒子"，软件装在里面不会搞乱你的电脑。

**第一步：安装 Docker**

macOS 用户：
```bash
# 打开终端，运行这条命令
brew install --cask docker
```

Windows 用户：
去 Docker 官网下载 Docker Desktop，安装包有点大（500MB+），耐心等待。

**第二步：创建配置文件夹**

```bash
mkdir -p ~/openclaw/{config,data,plugins}
cd ~/openclaw
```

**第三步：创建配置文件**

新建一个 `config.yaml` 文件，内容如下：

```yaml
llm:
  provider: "anthropic"          # 用 Claude
  model: "claude-sonnet-4-20250514"
  api_key: "你的API密钥"          # 去 Anthropic 官网申请

channels:
  telegram:
    enabled: false                # 先不接平台，后面再说
```

**第四步：启动！**

```bash
docker run -d \
  --name openclaw \
  -p 18789:18789 \
  -v ~/openclaw/config:/app/config \
  openclaw/openclaw:latest
```

**第五步：验证是否成功**

```bash
# 检查容器状态
docker ps

# 查看日志
docker logs openclaw
```

看到类似这样的输出就成功了：
```
Gateway listening on port 18789
```

---

### 方式二：npm 安装（不需要 Docker）

适合不想装 Docker 的同学。

**第一步：安装 Node.js**

macOS：
```bash
# 用 Homebrew 安装
brew install node@22
```

Windows：去 Node.js 官网下载安装包。

**第二步：安装 OpenClaw**

```bash
# 全局安装
npm install -g openclaw@latest

# 验证安装成功
openclaw --version
```

**第三步：引导配置**

```bash
# 启动引导程序
openclaw onboard

# 它会问你一些问题，一路回车就行
```

**第四步：启动服务**

```bash
# 启动
openclaw gateway

# 或者后台运行
openclaw gateway &
```

访问 `http://127.0.0.1:18789/` 看看能不能打开。

---

### 安装后第一件事：接 Telegram

> 如果你没有 Telegram Bot 的 Token，先去 [@BotFather](https://t.me/botfather) 创建一个。

**编辑配置文件 `config.yaml`**：

```yaml
llm:
  provider: "anthropic"
  model: "claude-sonnet-4-20250514"
  api_key: "sk-ant-xxxxx"       # 你的 Claude API Key

channels:
  telegram:
    enabled: true
    bot_token: "123456:ABC-xxxxx"  # BotFather 给你的 Token
    allowed_users:
      - 123456789                  # 你的 Telegram 用户 ID
```

**重启服务**：

```bash
# Docker 方式
docker restart openclaw

# npm 方式
# Ctrl+C 停止，然后重新运行 openclaw gateway
```

**给你的 Bot 发条消息试试！**

打开 Telegram，找到你的 Bot，点击 Start，然后随便发句话。

> 如果 Bot 没反应，先检查日志：`docker logs openclaw`

## 常见问题 Q&A

### Q: 报"Module not found"错误？

**原因**：依赖没装好。

**解决**：
```bash
npm install -g openclaw@latest
```

或者重建 Docker 镜像：
```bash
docker rm openclaw
docker rmi openclaw/openclaw:latest
docker pull openclaw/openclaw:latest
# 然后重新运行
```

### Q: Bot 不回复消息？

**排查步骤**：

1. 检查 Bot Token 是否正确
2. 检查 `allowed_users` 是否包含你的用户 ID
3. 检查 API Key 是否有效
4. 查看日志找错误信息：

```bash
docker logs openclaw 2>&1 | grep -i error
```

### Q: 消息发出去没反应，一直转圈？

通常是 AI 模型那边出问题了，比如：

- API Key 过期/没钱了
- 网络连接失败
- 模型服务不可用

**快速诊断**：
```bash
# 测试 API 是否能连通
curl -X POST https://api.anthropic.com/v1/messages \
  -H "x-api-key: 你的API_KEY" \
  -H "anthropic-version: 2023-06-01" \
  -d '{"model":"claude-sonnet-4-20250514","max_tokens":10,"messages":[{"role":"user","content":"hi"}]}'
```

### Q: 想用中文但 Bot 回复英文？

在系统提示词里告诉它用中文。编辑配置：

```yaml
system:
  prompt: "你是一个友好的中文AI助手，请始终用中文回答问题。"
```

## 小龙虾生态 vs 其他工具

很多人会问：这和 Dify、Coze、n8n 有什么区别？

| 工具 | 定位 | 小龙虾适合你吗？ |
|------|------|-----------------|
| **Dify** | 低代码 AI 应用平台 | 需要可视化搭建？选 Dify |
| **Coze** | 字节跳动的 Bot 平台 | 不想自己部署？选 Coze |
| **n8n** | 工作流自动化 | 只是要自动化？选 n8n |
| **OpenClaw** | 个人 AI 助手 | 想完全控制、数据隐私优先？选它 |

> 简单说：Coze/Dify 是"租房子"，OpenClaw 是"买房子"——买房子贵点但完全是你自己的。
