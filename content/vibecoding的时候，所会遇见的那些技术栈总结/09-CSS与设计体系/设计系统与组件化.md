# 设计系统与组件化

> 本文档深入探讨现代前端开发中设计系统与组件化的核心概念，从 Design Token 的三层架构到样式隔离方案，从主题切换机制到最佳实践，全方位解析构建可维护、高复用 UI 的工程化路径。

## 技术概述与定位

### 设计系统的定义与价值

设计系统不仅仅是一组 UI 组件的集合，它是一套完整的设计语言、工程规范和协作流程的体系。一个成熟的设计系统能够将设计师和开发者的协作效率提升数倍，同时确保产品在不同平台和场景下保持视觉和交互的一致性。

设计系统的核心价值体现在多个维度。首先是效率提升，通过预定义的组件和样式，开发团队无需从零开始构建每一个 UI 元素，可以直接复用经过设计和测试验证的组件。其次是一致性保障，当所有产品都基于同一套设计系统构建时，视觉风格、交互模式和行为预期都会保持高度一致，用户体验更加连贯。第三是可维护性增强，集中管理的设计规范使得样式修改和功能更新可以在一个地方完成，影响会自动传播到所有使用该系统的产品。

设计系统的演进经历了几个重要阶段。早期的设计系统主要是静态的设计规范文档，包含颜色代码、字体大小等视觉属性。随着前端工程化的发展，设计系统开始包含可复用的代码组件。而今天的设计系统已经演变为包含设计资产、代码组件、设计原则、交互模式、文档站点等多维度内容的完整体系，并且能够实现设计与代码的同步更新。

现代设计系统的典型架构包含以下核心组成部分：设计 Token 作为最底层的视觉变量，定义了颜色、字体、间距等基础属性；原子组件如按钮、输入框、图标等构成最小功能单元；分子组件由原子组件组合而成，实现更复杂的交互功能；有机组件或模板则是页面级别的布局模式；最后是文档系统，为设计者和开发者提供清晰的使用指南和示例代码。

### 组件化的核心概念

组件化是现代前端开发的基础范式之一，它将用户界面拆分为独立、可复用、可组合的部件。每个组件封装了自己的结构、样式和行为，对外提供明确的接口，内部实现则对使用者透明。这种封装性使得组件可以在不同项目中复用，也使得大型前端项目的开发和维护变得更加可控。

组件化的分层策略对于构建可维护的系统至关重要。在原子设计方法论中，组件被分为原子、分子、有机体、模板和页面五个层级。原子是最基础的 UI 元素，如按钮、标签、输入框；分子是由多个原子组合而成的简单功能单元，如搜索框（由输入框和按钮组成）；有机体是更复杂的界面片段，可以独立完成特定功能；模板定义了页面的布局结构；页面则是模板的具体实例。

组件的接口设计决定了其可用性和灵活性。良好的组件接口应该足够通用以适应多种场景，同时又要足够具体以提供明确的使用指导。属性（Props）作为组件的输入参数，应该具有合理的默认值、清晰的类型定义和必要的文档说明。组件的事件接口定义了它可以通知外部的行为，而状态接口则表明组件可能处于的内部状态。

组件的变体处理是接口设计中的重要课题。通过变体机制，一个组件可以适应不同的视觉风格和功能需求。常见的变体处理方式包括：通过属性控制不同的视觉表现、通过组合实现功能扩展、通过继承实现主题定制等。变体的设计需要在灵活性和复杂度之间取得平衡，避免过度设计导致的维护负担。

## 核心概念详解

### Design Token 三层架构详解

Design Token（设计令牌）是设计系统中存储设计决策的最小单位，它们以**键值对**的形式将视觉设计属性（颜色、字体、间距、阴影等）抽象为可复用的变量。在组件化开发中，Design Token 扮演着连接设计与工程的桥梁角色。

### 1.1 三层架构模型

现代 Design Token 采用**全局 Token → 语义 Token → 组件 Token**的三层架构，这种分层策略实现了设计一致性与灵活性的完美平衡。

#### 第一层：全局 Token（Primitives）

全局 Token 是设计系统中最原始的值，它们直接对应 Figma/Sketch 等设计工具中的具体数值。这层 Token 命名应清晰表达其视觉属性。

```css
/* 全局颜色 Token */
--color-blue-50: #eff6ff;
--color-blue-100: #dbeafe;
--color-blue-200: #bfdbfe;
--color-blue-300: #93c5fd;
--color-blue-400: #60a5fa;
--color-blue-500: #3b82f6;
--color-blue-600: #2563eb;
--color-blue-700: #1d4ed8;
--color-blue-800: #1e40af;
--color-blue-900: #1e3a8a;

/* 全局间距 Token */
--spacing-0: 0;
--spacing-1: 0.25rem;   /* 4px */
--spacing-2: 0.5rem;    /* 8px */
--spacing-3: 0.75rem;    /* 12px */
--spacing-4: 1rem;       /* 16px */
--spacing-6: 1.5rem;    /* 24px */
--spacing-8: 2rem;       /* 32px */
--spacing-12: 3rem;      /* 48px */
--spacing-16: 4rem;      /* 64px */

/* 全局圆角 Token */
--radius-sm: 0.25rem;
--radius-md: 0.5rem;
--radius-lg: 0.75rem;
--radius-xl: 1rem;
--radius-full: 9999px;
```

#### 第二层：语义 Token（Semantic Tokens）

语义 Token 是对全局 Token 的抽象，它们表达**含义**而非具体值。通过语义 Token，同一视觉属性可以在不同上下文中复用不同值。

```css
/* 语义颜色 Token */
:root {
  /* 品牌色 */
  --color-brand-primary: var(--color-blue-600);
  --color-brand-secondary: var(--color-violet-600);
  
  /* 背景色 */
  --color-background-base: #ffffff;
  --color-background-raised: #f8fafc;
  --color-background-sunken: #f1f5f9;
  --color-background-overlay: rgba(0, 0, 0, 0.5);
  
  /* 文本色 */
  --color-text-primary: #0f172a;
  --color-text-secondary: #475569;
  --color-text-tertiary: #94a3b8;
  --color-text-inverse: #ffffff;
  
  /* 边框色 */
  --color-border-default: #e2e8f0;
  --color-border-subtle: #f1f5f9;
  --color-border-strong: #cbd5e1;
  --color-border-focus: var(--color-blue-500);
  
  /* 功能色 */
  --color-success: #10b981;
  --color-warning: #f59e0b;
  --color-error: #ef4444;
  --color-info: #3b82f6;
}

/* 暗色主题语义 Token */
[data-theme="dark"] {
  --color-background-base: #0f172a;
  --color-background-raised: #1e293b;
  --color-background-sunken: #020617;
  
  --color-text-primary: #f8fafc;
  --color-text-secondary: #cbd5e1;
  --color-text-tertiary: #64748b;
  
  --color-border-default: #334155;
  --color-border-subtle: #1e293b;
  --color-border-strong: #475569;
}
```

#### 第三层：组件 Token（Component Tokens）

组件 Token 是针对特定组件的 Token，它们继承或覆盖语义 Token，为组件提供精确的样式控制。

```css
/* Button 组件 Token */
:root {
  /* 按钮尺寸 */
  --button-padding-x-sm: 0.75rem;
  --button-padding-x-md: 1rem;
  --button-padding-x-lg: 1.5rem;
  --button-padding-y-sm: 0.375rem;
  --button-padding-y-md: 0.5rem;
  --button-padding-y-lg: 0.75rem;
  
  /* 按钮圆角 */
  --button-border-radius: var(--radius-md);
  
  /* 按钮字体 */
  --button-font-weight: 500;
  --button-font-size-sm: 0.875rem;
  --button-font-size-md: 0.9375rem;
  --button-font-size-lg: 1.125rem;
  
  /* 按钮变体 */
  --button-primary-bg: var(--color-brand-primary);
  --button-primary-color: var(--color-text-inverse);
  --button-primary-hover-bg: var(--color-blue-700);
  
  --button-secondary-bg: transparent;
  --button-secondary-color: var(--color-text-primary);
  --button-secondary-border: var(--color-border-default);
  
  --button-ghost-hover-bg: var(--color-background-raised);
}
```

## 2. 组件化样式隔离方案对比

样式隔离是组件化的核心挑战。当多个组件共存于同一页面时，样式冲突会导致意外渲染。业界发展出多种解决方案，各有优劣。

### 组件化样式隔离方案对比

样式隔离是组件化的核心挑战。当多个组件共存于同一页面时，样式冲突会导致意外渲染。业界发展出多种解决方案，各有优劣。

#### CSS Modules 深度实践

CSS Modules 通过文件级别的唯一类名生成实现样式隔离，是 React 生态中最广泛使用的方案之一。

```css
/* Button.module.css */
.button {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  padding: var(--button-padding-y-md) var(--button-padding-x-md);
  font-size: var(--button-font-size-md);
  font-weight: var(--button-font-weight);
  border-radius: var(--button-border-radius);
  transition: all 0.15s ease-in-out;
}

.button:disabled {
  opacity: 0.5;
  cursor: not-allowed;
}

.primary {
  background-color: var(--button-primary-bg);
  color: var(--button-primary-color);
}

.primary:hover:not(:disabled) {
  background-color: var(--button-primary-hover-bg);
}

.secondary {
  background-color: var(--button-secondary-bg);
  color: var(--button-secondary-color);
  border: 1px solid var(--button-secondary-border);
}

.small {
  padding: var(--button-padding-y-sm) var(--button-padding-x-sm);
  font-size: var(--button-font-size-sm);
}

.large {
  padding: var(--button-padding-y-lg) var(--button-padding-x-lg);
  font-size: var(--button-font-size-lg);
}
```

```tsx
// Button.tsx
import styles from './Button.module.css';

interface ButtonProps {
  variant?: 'primary' | 'secondary';
  size?: 'small' | 'medium' | 'large';
  disabled?: boolean;
  children: React.ReactNode;
  onClick?: () => void;
}

export function Button({
  variant = 'primary',
  size = 'medium',
  disabled = false,
  children,
  onClick
}: ButtonProps) {
  const classNames = [
    styles.button,
    styles[variant],
    size !== 'medium' && styles[size]
  ].filter(Boolean).join(' ');

  return (
    <button 
      className={classNames} 
      disabled={disabled}
      onClick={onClick}
    >
      {children}
    </button>
  );
}
```

**编译后的类名**：`Button_button__3x7kP`

> [!TIP] CSS Modules 优势
> - 原生 CSS，无需学习新语法
> - 与现有 CSS 工具链完全兼容
> - 编译时隔离，无运行时开销

> [!WARNING] CSS Modules 局限
> - 类名组合需要手动处理
> - 无法实现真正的动态样式（如主题变量）
> - 跨组件样式复用需要额外配置

### 2.2 Styled Components

Styled Components 是**CSS-in-JS** 方案的典型代表，通过标签模板字符串和 JavaScript 运行时实现样式定义。

```tsx
import styled, { css, createGlobalStyle } from 'styled-components';

// 定义主题类型
interface Theme {
  colors: {
    primary: string;
    secondary: string;
    background: string;
    text: string;
  };
  spacing: {
    sm: string;
    md: string;
    lg: string;
  };
}

// 通用按钮组件
const Button = styled.button<{ $variant?: 'primary' | 'secondary'; $size?: 'sm' | 'md' | 'lg' }>`
  display: inline-flex;
  align-items: center;
  justify-content: center;
  font-weight: 500;
  border-radius: 0.5rem;
  transition: all 0.15s ease-in-out;
  cursor: pointer;
  border: none;
  outline: none;

  /* 尺寸变体 */
  ${({ $size = 'md' }) => {
    const sizes = {
      sm: css`
        padding: 0.375rem 0.75rem;
        font-size: 0.875rem;
      `,
      md: css`
        padding: 0.5rem 1rem;
        font-size: 0.9375rem;
      `,
      lg: css`
        padding: 0.75rem 1.5rem;
        font-size: 1.125rem;
      `
    };
    return sizes[$size];
  }}

  /* 颜色变体 */
  ${({ $variant = 'primary', theme }) => {
    const variants = {
      primary: css`
        background-color: ${theme.colors.primary};
        color: white;
        &:hover:not(:disabled) {
          filter: brightness(1.1);
        }
      `,
      secondary: css`
        background-color: transparent;
        color: ${theme.colors.text};
        border: 1px solid ${theme.colors.secondary};
        &:hover:not(:disabled) {
          background-color: ${theme.colors.secondary}20;
        }
      `
    };
    return variants[$variant];
  }}

  &:disabled {
    opacity: 0.5;
    cursor: not-allowed;
  }
`;

// 使用示例
function App() {
  return (
    <Button $variant="primary" $size="md">
      Click Me
    </Button>
  );
}
```

> [!TIP] Styled Components 优势
> - 真正的动态样式，主题切换零配置
> - 组件与样式共处一地，维护成本低
> - 自动添加唯一类名，样式隔离完善

> [!WARNING] Styled Components 局限
> - 运行时开销，增加 bundle size
> - SSR 需要额外配置（styled-components SSR）
> - IDE 类型提示支持依赖插件

### 2.3 Tailwind CSS

Tailwind 采用**原子化 CSS** 理念，将样式抽象为单一职责的工具类，通过组合实现复杂 UI。

```html
<!-- 基础按钮 -->
<button class="
  inline-flex items-center justify-center
  px-4 py-2
  text-sm font-medium
  rounded-md
  transition-colors duration-150
  bg-blue-600 text-white
  hover:bg-blue-700
  disabled:opacity-50 disabled:cursor-not-allowed
">
  Primary Button
</button>

<!-- 次要按钮 -->
<button class="
  inline-flex items-center justify-center
  px-4 py-2
  text-sm font-medium
  rounded-md
  border border-gray-300
  bg-white text-gray-700
  hover:bg-gray-50
">
  Secondary Button
</button>
```

通过 `@apply` 指令可以将重复组合提取为自定义类：

```css
/* 在 CSS 文件中 */
```

```tsx
// ThemeProvider.tsx
import React, { createContext, useContext, useState, useEffect } from 'react';

type Theme = 'light' | 'dark';

interface ThemeContextValue {
  theme: Theme;
  toggleTheme: () => void;
  setTheme: (theme: Theme) => void;
}

const ThemeContext = createContext<ThemeContextValue | undefined>(undefined);

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setThemeState] = useState<Theme>(() => {
    // 初始化时检查 localStorage
    const stored = localStorage.getItem('theme') as Theme | null;
    if (stored) return stored;
    
    // 检查系统偏好
    if (window.matchMedia('(prefers-color-scheme: dark)').matches) {
      return 'dark';
    }
    return 'light';
  });

  useEffect(() => {
    const root = document.documentElement;
    
    // 设置 data-theme 属性
    root.setAttribute('data-theme', theme);
    
    // 保存到 localStorage
    localStorage.setItem('theme', theme);
    
    // 更新 meta theme-color
    const metaThemeColor = document.querySelector('meta[name="theme-color"]');
    if (metaThemeColor) {
      metaThemeColor.setAttribute(
        'content', 
        theme === 'dark' ? '#0f172a' : '#ffffff'
      );
    }
  }, [theme]);

  const toggleTheme = () => {
    setThemeState(prev => prev === 'light' ? 'dark' : 'light');
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme, setTheme: setThemeState }}>
      {children}
    </ThemeContext.Provider>
  );
}

export function useTheme() {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error('useTheme must be used within a ThemeProvider');
  }
  return context;
}
```

### 3.2 主题色系统

品牌主题色需要支持动态调整，常用方案是 HSL 颜色空间的操作。

```typescript
// theme-color.ts
/**
 * 基于 HSL 的主题色生成系统
 * 通过调整 HSL 的 L 值生成同色系的浅色变体
 */
export interface ThemeColors {
  primary: string;
  primaryLight: string;
  primaryDark: string;
  accent: string;
}

export function generateThemeColors(baseColor: string): ThemeColors {
  const hsl = hexToHSL(baseColor);
  
  return {
    primary: `hsl(${hsl.h}, ${hsl.s}%, ${hsl.l}%)`,
    primaryLight: `hsl(${hsl.h}, ${hsl.s}%, ${Math.min(hsl.l + 20, 95)}%)`,
    primaryDark: `hsl(${hsl.h}, ${hsl.s}%, ${Math.max(hsl.l - 15, 10)}%)`,
    accent: `hsl(${(hsl.h + 30) % 360}, ${hsl.s}%, ${hsl.l}%)`
  };
}

export function hexToHSL(hex: string): { h: number; s: number; l: number } {
  // 移除 # 号
  hex = hex.replace('#', '');
  
  // 解析 RGB
  const r = parseInt(hex.substring(0, 2), 16) / 255;
  const g = parseInt(hex.substring(2, 4), 16) / 255;
  const b = parseInt(hex.substring(4, 6), 16) / 255;
  
  const max = Math.max(r, g, b);
  const min = Math.min(r, g, b);
  let h = 0;
  let s = 0;
  const l = (max + min) / 2;
  
  if (max !== min) {
    const d = max - min;
    s = l > 0.5 ? d / (2 - max - min) : d / (max + min);
    
    switch (max) {
      case r:
        h = ((g - b) / d + (g < b ? 6 : 0)) / 6;
        break;
      case g:
        h = ((b - r) / d + 2) / 6;
        break;
      case b:
        h = ((r - g) / d + 4) / 6;
        break;
    }
  }
  
  return {
    h: Math.round(h * 360),
    s: Math.round(s * 100),
    l: Math.round(l * 100)
  };
}

// 应用主题色
export function applyThemeColor(color: string) {
  const colors = generateThemeColors(color);
  const root = document.documentElement;
  
  root.style.setProperty('--theme-primary', colors.primary);
  root.style.setProperty('--theme-primary-light', colors.primaryLight);
  root.style.setProperty('--theme-primary-dark', colors.primaryDark);
  root.style.setProperty('--theme-accent', colors.accent);
}
```

```tsx
interface SelectContextValue {
  value: string;
  onChange: (value: string) => void;
  open: boolean;
  setOpen: (open: boolean) => void;
}

const SelectContext = createContext<SelectContextValue | null>(null);

const useSelect = () => {
  const context = useContext(SelectContext);
  if (!context) {
    throw new Error('Select components must be used within Select');
  }
  return context;
};

const Select = ({
  value,
  onChange,
  children
}: {
  value: string;
  onChange: (value: string) => void;
  children: ReactNode;
}) => {
  const [open, setOpen] = useState(false);

  return (
    <SelectContext.Provider value={{ value, onChange, open, setOpen }}>
      <div className="select-container">
        {children}
      </div>
    </SelectContext.Provider>
  );
};

// 触发器组件
const SelectTrigger = ({
  children,
  className = ''
}: {
  children: ReactNode;
  className?: string;
}) => {
  const { open, setOpen, value } = useSelect();

  return (
    <button
      type="button"
      className={`select-trigger ${className}`}
      onClick={() => setOpen(!open)}
      aria-expanded={open}
      aria-haspopup="listbox"
    >
      {children || value}
      <span className="select-arrow">{open ? '▲' : '▼'}</span>
    </button>
  );
};

// 选项列表组件
const SelectOptions = ({
  children,
  className = ''
}: {
  children: ReactNode;
  className?: string;
}) => {
  const { open } = useSelect();

  if (!open) return null;

  return (
    <ul
      className={`select-options ${className}`}
      role="listbox"
      aria-orientation="vertical"
    >
      {children}
    </ul>
  );
};

// 选项组件
const SelectOption = ({
  value,
  children,
  disabled = false,
  className = ''
}: {
  value: string;
  children: ReactNode;
  disabled?: boolean;
  className?: string;
}) => {
  const { value: selectedValue, onChange, setOpen } = useSelect();
  const isSelected = value === selectedValue;

    if (!disabled) {
      onChange(value);
      setOpen(false);
    }

      role="option"
      aria-selected={isSelected}
      aria-disabled={disabled}
      onClick={handleClick}

      {children}
      {isSelected && <span className="check-icon">✓</span>}
    </li>
  );
};
```

```typescript
// token-generator.ts
interface DesignToken {
  name: string;
  value: string;
  description?: string;
  category: string;
  subCategory?: string;
}

class TokenGenerator {
  private tokens: DesignToken[] = [];

  addToken(token: DesignToken) {
    this.tokens.push(token);
    return this;
  }

  // 生成 CSS 变量
  generateCSS(): string {
    const cssVars = this.tokens.map(token => {
      const varName = token.name.replace(/([A-Z])/g, '-$1').toLowerCase();
      return `  --${varName}: ${token.value};`;
    });

    return `:root {\n${cssVars.join('\n')}\n}`;
  }

  // 生成 TypeScript 类型
  generateTypes(): string {
    const tokensByCategory = this.groupByCategory();

    const typeDefs = Object.entries(tokensByCategory).map(([category, tokens]) => {
      const typeName = this.capitalize(category);
      const values = tokens.map(t => `  ${t.name}: string;`).join('\n');
      return `export const ${typeName}Tokens = {\n${values}\n} as const;`;
    });

    return typeDefs.join('\n\n');
  }

  // 生成 Tailwind 配置
  generateTailwindConfig(): object {
    const colors = this.tokens
      .filter(t => t.category === 'color')
      .reduce((acc, t) => {
        const name = t.name.replace(/([A-Z])/g, '-$1').toLowerCase();
        acc[name] = `var(--${name.replace(/([A-Z])/g, '-$1').toLowerCase()})`;
        return acc;
      }, {} as Record<string, string>);

    return { theme: { extend: { colors } } };
  }

  private groupByCategory() {
    return this.tokens.reduce((acc, token) => {
      if (!acc[token.category]) {
        acc[token.category] = [];
      }
      acc[token.category].push(token);
      return acc;
    }, {} as Record<string, DesignToken[]>);
  }

  private capitalize(str: string): string {
    return str.charAt(0).toUpperCase() + str.slice(1);
  }
}

// 使用示例
const tokens = new TokenGenerator()
  .addToken({ name: 'primary', value: '#3b82f6', category: 'color' })
  .addToken({ name: 'secondary', value: '#8b5cf6', category: 'color' })
  .addToken({ name: 'sm', value: '0.25rem', category: 'spacing' })
  .addToken({ name: 'md', value: '0.5rem', category: 'spacing' })
  .addToken({ name: 'base', value: '1rem', category: 'fontSize' });

console.log(tokens.generateCSS());
```

```typescript
    return `import type { Meta, StoryObj } from '@storybook/react';

const meta: Meta<typeof ${meta.name}> = {
  title: 'Components/${meta.name}',
  component: ${meta.name},
  tags: ['autodocs'],

        component: '${meta.description}',
      },
    },
  },

};

export default meta;
type Story = StoryObj<typeof ${meta.name}>;

${this.generateStories(meta)}
`;
  }

  private generateArgTypes(meta: ComponentMeta): string {
    const argTypes = meta.props.map(prop => {
      const control = this.getControlForType(prop.type);
      return `      ${prop.name}: {
        description: '${prop.description}',
        control: { type: '${control}' },
      }`;

${argTypes}
  },`;
  }

  private getControlForType(type: string): string {
    if (type === 'boolean') return 'boolean';
    if (type === 'string') return 'text';
    if (type === 'number') return 'number';
    if (type === 'select') return 'select';
    return 'object';
  }

  private generateStories(meta: ComponentMeta): string {
    if (!meta.variants || meta.variants.length === 0) {
      return `export const Default: Story = {
  args: {
${meta.props.map(p => `    ${p.name}: ${this.getDefaultValue(p)},`).join('\n')}
  },
};`;
    }

    return meta.variants.map(variant => `
export const ${this.capitalize(variant.name)}: Story = {
  args: {
${variant.values.map(v => `    ${v}: true,`).join('\n')}
  },
  parameters: {
    docs: {
      description: {
        story: '${variant.description}',
      },
    },
  },
};`).join('\n');
  }

  private getDefaultValue(prop: PropMeta): string {
    if (prop.defaultValue) return prop.defaultValue;
    if (prop.type === 'boolean') return 'false';
    if (prop.type === 'string') return "'Default'";
    if (prop.type === 'number') return '0';
    return 'undefined';
  }

  private capitalize(str: string): string {
    return str.charAt(0).toUpperCase() + str.slice(1);
  }
```

## 设计系统与组件化

### 组件设计原则

组件设计需要遵循一系列原则以确保代码的可维护性和可扩展性。

#### 单一职责原则

每个组件应该只负责一个功能或关注点。这使得组件更容易理解、测试和复用。当一个组件承担太多职责时，它会变得复杂且难以维护。

```typescript
// 好的实践：职责分离
// 按钮组件只负责按钮的交互
const Button = ({ onClick, children, variant }) => (
  <button className={`btn btn-${variant}`} onClick={onClick}>
    {children}
  </button>
);

// 图标组件只负责图标显示
const Icon = ({ name, size, color }) => (
  <svg className={`icon icon-${name}`} width={size} height={size} fill={color}>
    <use href={`#${name}`} />
  </svg>
);

// 组合使用
const DeleteButton = ({ onDelete }) => (
  <Button onClick={onDelete} variant="danger">
    <Icon name="trash" size={16} />
    删除
  </Button>
);

// 差的实践：职责混合
const BadButton = ({ onDelete, icon, text, onHover, onClick }) => (
  <button
    onClick={onClick}
    onMouseEnter={onHover}
    className="btn"
  >
    <img src={icon} />
    {text}
    {onDelete && <span onClick={onDelete}>x</span>}
  </button>
);
```

#### 开闭原则

组件应该对扩展开放，对修改关闭。通过适当的抽象和组合，组件可以在不修改源代码的情况下适应新的需求。


#### 里氏替换原则

子类型必须能够替换其基类型，而不改变程序的正确性。这意味着组件的接口应该保持一致性，子组件应该能够替代父组件使用。

```typescript
// 定义基础接口
interface FormFieldProps {
  label: string;
  error?: string;
  required?: boolean;
  disabled?: boolean;
}

// 所有表单字段都遵循相同接口
const TextField = ({ label, error, ...props }: FormFieldProps) => (
  <div className="form-field">
    <label>{label}</label>
    <input {...props} />
    {error && <span className="error">{error}</span>}
  </div>
);

const SelectField = ({ label, options, error, ...props }: FormFieldProps) => (
  <div className="form-field">
    <label>{label}</label>
    <select {...props}>
      {options.map(opt => <option key={opt.value} value={opt.value}>{opt.label}</option>)}
    </select>
    {error && <span className="error">{error}</span>}
  </div>
);

const TextareaField = ({ label, error, ...props }: FormFieldProps) => (
  <div className="form-field">
    <label>{label}</label>
    <textarea {...props} />
    {error && <span className="error">{error}</span>}
  </div>
);

// 统一的表单字段渲染器
const FormField = ({ type, ...props }) => {
  const components = { text: TextField, select: SelectField, textarea: TextareaField };
  const Component = components[type] || TextField;
  return <Component {...props} />;
};
```

#### 依赖倒置原则

高层次的模块不应该依赖低层次的模块，两者都应该依赖抽象。

```typescript
// 定义抽象接口
interface DataService {
  fetch<T>(url: string): Promise<T>;
  post<T>(url: string, data: unknown): Promise<T>;
}

// 低层次实现
class HttpDataService implements DataService {
  async fetch<T>(url: string): Promise<T> {
    const response = await fetch(url);
    return response.json();
  }

  async post<T>(url: string, data: unknown): Promise<T> {
    const response = await fetch(url, {
      method: 'POST',
      body: JSON.stringify(data),
    });
    return response.json();
  }
}

// 高层次组件依赖抽象而非具体实现
const UserList = ({ dataService }: { dataService: DataService }) => {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    dataService.fetch<User[]>('/api/users').then(setUsers);
  }, []);

  return (
    <ul>
      {users.map(user => <li key={user.id}>{user.name}</li>)}
    </ul>
  );
};

// 使用时注入具体实现
const App = () => (
  <UserList dataService={new HttpDataService()} />
);
```

```typescript
  loading?: boolean;
  leftIcon?: React.ReactNode;
  rightIcon?: React.ReactNode;
  children: React.ReactNode;
  onClick?: (event: React.MouseEvent<HTMLButtonElement>) => void;
  type?: 'button' | 'submit' | 'reset';
  className?: string;
}

// 2. 扩展原生属性
interface ExtendedButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary';
  size?: 'sm' | 'md' | 'lg';
}

// 3. as 模式（Polymorphic Components）
type ButtonBaseProps<T extends React.ElementType> = {
  as?: T;
} & React.ComponentPropsWithoutRef<T>;

const Button = <T extends React.ElementType = 'button'>(
  props: ButtonBaseProps<T>
) => {
  const { as: Component = 'button', ...rest } = props;
  return <Component {...rest} />;
};

// 使用
<Button as="a" href="/home">Link Button</Button>
<Button as={Link} to="/home">Router Link</Button>

// 4. forwardRef 模式
const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ variant = 'primary', children, ...props }, ref) => {
    return (
      <button ref={ref} className={`btn-${variant}`} {...props}>
        {children}
      </button>
    );
  }
);

Button.displayName = 'Button';

// 5. 联合 Props 模式
interface BaseProps {
  variant?: 'default' | 'outline';
}

interface LinkProps extends BaseProps {
  href: string;
  external?: boolean;
}

interface ButtonProps extends BaseProps {
  onClick?: () => void;
  type?: 'button' | 'submit';
}

type Props = LinkProps | ButtonProps;
```

## 常见问题与解决方案

### 问题 1：组件发布后样式不生效

**症状**：发布的 npm 包样式在消费项目不生效

**原因**：CSS 文件未正确打包或导入

**解决方案**：

```bash
# 检查 package.json 的导出配置
{
  "main": "dist/index.js",
  "module": "dist/index.esm.js",
  "types": "dist/index.d.ts",
  "exports": {
    ".": {
      "import": "./dist/index.esm.js",
      "require": "./dist/index.js",
      "types": "./dist/index.d.ts"
    },
    "./dist/style.css": "./dist/style.css"
  },
  "files": [
    "dist"
  ]
}

# 确保 CSS 文件被复制到 dist
# rollup.config.js
export default {
  // ...
  plugins: [
    postcss({
      extract: true,  // 提取 CSS 到单独文件
      minimize: true,
      modules: {
        generateScopedName: '[name]__[local]'
      }
    })
  ]
};
```

### 问题 2：主题切换闪烁

**症状**：页面加载时主题样式闪烁

**解决方案**：

```html
<!-- 1. 在 HTML head 中添加主题检测脚本 -->
<head>
  <script>
    (function() {
      // 检测 localStorage 或系统偏好
      const theme = localStorage.getItem('theme') || 
        (window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');
      
      // 同步设置到 HTML 元素
      document.documentElement.setAttribute('data-theme', theme);
    })();
  </script>
  
  <!-- 2. 添加阻塞样式确保无闪烁 -->
  <style>
    html:not([data-theme]) {
      visibility: hidden;
    }
  </style>
</head>

<!-- 3. 在 body 底部恢复可见 -->
<body>
  <div id="app">
    <script>
      document.documentElement.removeAttribute('data-theming-loading');
      document.documentElement.style.visibility = 'visible';
    </script>
  </div>
</body>
```
